INSTANCIER UN PROJET LIBRE-SERVICE

Contexte d'utilisation​

Une ligne d'affaires désire produire elle-même ses rapports et les publier à plusieurs utilisateurs. La volatilité des besoins ou la nécessité de modifier ou de faire évoluer les rapports est importante. La fréquence des changements, de l'évolution ou de l'ajout de nouveaux rapports demande une liberté et une indépendance de la part de la ligne d'affaires versus l'équipe de développement BI corporatif pour la création des rapports.

La ligne d'affaires dispose de ressources possédant les compétences de super-utilisateurs ou désirant monter en compétence pour remplir ce rôle.

Scénario d'utilisation (description)​

Un jeu de données créer par l'équipe de développement BI corporative sera mis à disposition des super utilisateurs. Aucun projet d'affaires corporatif n'est en place. L'équipe de développement corporatif ne produit pas de rapport pour ce projet.

Règles / prérequis​
  • Aucun rapport ne sera produit par l'équipe de développement BI corporatif.
  • Un seul sujet d'affaires.
  • La cible est la création d'un jeu de données et d'un environnement libre-service.

Type de projet possible​

  • Projet libre-service

Composants requis pour instancier la solution​

Le libre-service est un ajout à un espace de travail WS-DS, espace de travail de jeu de données. Voir [Composants requis pour instancier les solutions par type de projets](about:Composants requis pour instancier les solutions par type de projets) pour plus de détails sur la configuration par défaut des paramètres de passerelle et des espaces de travail de jeu de données.

Voir [Composants requis pour instancier l'option libre-service](about:[#####Composants requis pour instancier l'option libre-service) pour plus de détail sur les composants de libre-service.

Utilisation des groupes et comptes AD

Utilisation des groupes et comptes AD

Afin de faire la relation entre l'architecture fonctionnelle et la sécurité mise en place dans le service Power BI, voici sur quelles entités Power BI chacun des types groupes et comptes AD peuvent être assignés.

Groupe de sécurité AD - Administrateur Power BI​

Un seul groupe d'administrateurs est créé, et toutes les personnes responsables d'administrer les paramètres de Power BI Service pour l'organisation en sont membres.

Ce groupe est configuré dans la console d'administration d'Office 365 et donne accès à la console d'administration du service Power BI.

Groupe de sécurité AD - Administrateur des passerelles (Gateways)​

Un seul groupe d'administrateurs des passerelles et toutes les personnes en charge d'administrer les Gateways de Power BI en sont membres.

Ce groupe est appliqué dans Power BI Service sur la console de gestion des passerelles pour donner accès à cette console.

Le groupe AD d'administrateur de passerelle (Gateway) est assigné sur la console de gestion des passerelles dans le service Power BI :

Compte de sécurité AD - Compte de passerelle Power BI (Gateway)​

Un compte de service de passerelles est créé par source de données.

Ces comptes sont utilisés dans la configuration de la source dans la console de gestion des passerelles et servent à accéder à la source de données.

Un compte de service de passerelle peut être assigné sur les entités Power BI et de bases de données suivant:

Groupe de sécurité AD - Développeur BI​

Normalement, un groupe de développeurs Power BI est créé par équipe de développement.

Ce groupe est assigné à toutes les entités Power BI sous la responsabilité de l'équipe de développement.

Pour des fins de support et d'accompagnement aux gens d'affaires, le groupe AD de développement est aussi assigné sur les espaces de travail de libre-service et d'exploration (conception affaires).

Ce groupe est assigné directement sur les entités via Power BI Service.

Un groupe AD de développeurs BI peut être assigné sur les entités Power BI suivant:

Groupe de sécurité AD - Application Power BI​

Un groupe application est créé par application Power BI.

Ce groupe est appliqué dans Power BI Service dans le menu de gestion de l'application dans l'espace de travail de rapport correspondant.

Un groupe AD applicatif peut être assigné sur les entités Power BI suivant:

Groupe de sécurité AD - Super-utilisateurs​

Un groupe de super-utilisateurs est créé par contexte d'accès.

Ce groupe est assigné directement sur les entités Power BI Service.

Un groupe AD de super-utilisateurs affaires peut être assigné sur les entités Power BI suivant:

Groupe de sécurité AD - Concepteurs affaires​

Un groupe de concepteurs affaires est créé par projet d'exploration ou de conception. Un même groupe peut être assigné sur plusieurs Espaces de travail d'exploration si tous les utilisateurs partagent les mêmes accès.

Ce groupe est assigné directement sur les entités Power BI Service.

Certification des jeux de données

Certification des jeux de données

La solution Power BI offre à une organisation la possibilité de certifier des jeux de données, rapports et applications. Cela permet à un utilisateur d'être sûr que le contenu répond aux normes de qualité d'Investissement Québec en ce qui a trait à un produit Power BI. Un contenu certifié peut ainsi être considéré comme fiable, faisant autorité et prêt à être utilisé par le groupe de consommateur du produit.

Certaines étapes sont ainsi nécessaires avant de pouvoir certifier un jeu de données.

Hypothèse : le RDM est utilisé pour normaliser les données, les contrôles de qualité (test unitaire, fonctionnel, etc.) sont faits (à renouveler lors des MEP).

Évaluation des données​

Pour effectuer une évaluation des données, une équipe composée d’analystes et de développeurs doit profiler et faire une analyse des données qui semblent extrêmes ou suspectes En regardant les valeurs de chacune des colonnes de chacune des tables, l’équipe peut ainsi détecter les données qui semblent être inhabituelles. La définition des paramètres de normalité est établie avec le Subject Matter Expert (SME) de la ligne d’affaires du produit créé ou encore avec les utilisateurs finaux du jeu de données. C’est ainsi qu’une matrice de contrôle des valeurs peut être produite et utilisée pour vérifier les données. Un ajout des données manquantes au MDM ou au RDM devrait être vérifié.

Exemple : si une entreprise indique dans un formulaire qu’elle ne possède pas d’employés, l’analyste doit voir avec le SME pour déterminer ce que la donnée signifie et si elle est valide ou doit être modifiée / exclue.

Catégorisation de la qualité​

Vient ensuite la catégorisation de la qualité avec les utilisateurs. Ce sont ces derniers qui produiront une matrice d’évaluation de la qualité sous les 4 catégories :

  1. Actualité (rafraîchissement)

  2. Exactitude référentielle (qualité des données)

  3. Validité (niveau de rejet)

  4. Exhaustivité (jeu sans données manquantes)

(Établir les niveaux de qualité nécessaires)

Chacune des catégories a un podium de qualité qui permet de savoir si le jeu de données est conforme ou non. Pour chacun des barèmes, un pourcentage permet d’associer un niveau de qualité réparti sous 4 podiums :

PodiumPourcentage associé
OR95 à 100%
ARGENT90 à 95 %
BRONZE85 à 90%
ACIER85% et moins

Un jeu de données Or est le seul à être certifié dans Power BI Service.

Exécution des tests​

Le développeur doit saisir la matrice de contrôle des valeurs (évaluation des données) dans le pilotage du jeu de données afin de pouvoir afficher la qualité des données directement dans le jeu de données. Cela permet d’accéder à l’historique des niveaux de certification du jeu de données.

Stocker les résultats​

Les résultats doivent être ajoutés dans la voute de données et dans le comptoir d’audit général pour qu’une vue globale de tous les jeux de données soit possible ainsi que l’historique associé à chacun des jeux de données.

Rafraîchir le rapport d’audit​

Le rapport d’audit comprenant tous les jeux de données certifiés doit être mis à jour avec le nouveau jeu de donnée et sa matrice de contrôle des valeurs qui se retrouvent dans la voute de données et dans le comptoir d’audit général. Cela permet d’avoir une vue générale sur les jeux de données et leur certification. Également, des alertes peuvent être intégrées pour obtenir l’intervention des SME et des utilisateurs affaires afin de rectifier la situation dans le jeu de données si un changement important est décelé dans les données.

Ajouter une page de certification​

Une page de certification doit être ajoutée dans les pages du jeu de données pour y insérer les informations pertinentes. C’est-à-dire la page d’actualité-rafraîchissement pour connaître les moments où sont mises à jour les données. La page de certification permet de démontrer le niveau de qualité du jeu de données.

L’ajout de pictogramme de podium de certification (or, argent, bronze et acier) permet ensuite de confirmer que la certification est active et d’informer l’usager de quel niveau de certification est le jeu de données.

Exécution des essais​

Une programmation de l’exécution des essais de régression à la fréquence des rafraichissements des données devrait être mis en place pour que la certification du jeu de données puisse se faire.

Certifier le rapport​

C’est donc lorsque le jeu de données a été considéré comme podium Or et que toutes les autres étapes ont été effectuées que le jeu peut devenir certifié Power BI sur Power BI Service.

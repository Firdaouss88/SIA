Plusieurs sujets d'affaires dans un jeu de données Power BI

PLUSIEURS SUJETS D'AFFAIRES DANS UN JEU DE DONNÉES POWER BI

Contexte d'utilisation​

Le projet implique de consommer dans le même rapports ou tableaux de bord des données provenant de plusieurs sujets d'affaires ou comptoirs de données différents.

Scénario d'utilisation (description)​

Au moins deux comptoirs de données seront intégrés dans un même jeu de données Power BI.

Les raisons peuvent être variées pour justifier les sources de données ne peuvent pas être intégrées dans un seul comptoir de données.

  • On ne veut pas mettre ces différents sujets dans le même comptoir.
    • Les sources portent sur des sujets d'affaires distincts normalement pas utilisés ensemble pour produire des rapports.
  • Les sujets d'affaires sont généralement consommés par des publics différents.
    • Accès au données différents, groupes AD distincts.
  • Des comptoirs de données séparés existent déjà en production.
Règles / prérequis​
  • Les informations de plusieurs comptoirs doivent être présentés dans le même rapport.
  • La cible est un seul jeu de données Power BI.
  • Un seul contexte de sécurité est présent.
    • Tous les utilisateurs ont accès à 100% des informations présentes dans le comptoir de données.

Type de projet possible​

  • Projet d'affaires.
  • Projet d'affaires avec libre-service.
  • Projet de libre-service autonome.
  • Projet d'exploration affaires.
  • Projet d'exploration de développement.

Composants requis pour instancier la solution​

  1. Source de données

    La seule différence avec la configuration par défaut est que le jeu de données Power BI sera créé à partir d'au moins 2 comptoirs de données.

    L'intégration des différents sujet d'affaires se fait dans le modèle de données Power BI.

    Si l'intégration des différents sujets d'affaires exige des transformations aux données, celles-ci peuvent être réalisées :

    • Dans la couche de vue des comptoirs.
    • Dans un comptoir spécifiquement dédié à l'intégration des différents sujets.
      • Autant que possible on évite cette option.
      • Si on doit le faire, on ne duplique pas les données dans le comptoir d'intégration.
        • On importe dans le comptoir d'intégration seulement les tables (fait, dimension) ayant nécessités des transformations pour être conformes.
        • On crée des vues sur les comptoirs sources dans le comptoir d'intégration pour les autres dimensions et tables de faits.

    Note:

    Le fichier Power BI contiendra une connexion sur chaque comptoir (source de données) utilisé.

    La configuration de passerelle dans le service Power BI doit être faite pour chacune de ces sources.

  2. Autres composants

    • Voir configuration par défaut: hyperlien à venir
    • Les autres composants utilisés ont les rôles par défaut :
      • Serveur de passerelle.
      • Paramètres de passerelles.
      • WS-DS, espace de travail de jeu de données.
      • Jeu de données Power BI.

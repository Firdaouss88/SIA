Introduction-PBI-Libre-service

Introduction au Libre service

L'objectif de cette formation est de données à utilisateur les outils de bases pour créer des rapports simples avec Power BI Desktop afin de les partager avec ses collègues en utilisant un jeux de données existant et déjà publier dans Power BI Service.

Générique​

Power BI Desktop est une application gratuite qui s’installe sur un ordinateur local et permet de se connecter à des données, de les transformer et de les visualiser.

Avec Power BI Desktop, vous pouvez vous connecter à plusieurs sources de données différentes et les combiner dans un modèle de données (ce qui s’appelle la modélisation). Ce modèle de données vous permet de créer des visuels et des collections de visuels que vous pouvez partager sous forme de rapports avec d’autres personnes au sein de votre organisation.

Les utilisations les plus courantes de Power BI Desktop :

  • Se connecter aux données
  • Transformer et nettoyer ces données pour créer un modèle de données
  • Créer des visuels, par exemple, des graphiques, qui donnent des représentations visuelles des données
  • Créer des rapports correspondant à des collections de visuels, sur une ou plusieurs pages de rapport
  • Partager des rapports avec d’autres utilisateurs à l’aide du service Power BI

Se connecter aux données​

La première étape à faire pour démarrer avec Power BI Desktop est de se connecter aux données.

Quand vous devez se connecter à un jeu de données existants pour créer vos rapports vous suivez les étapes suivantes(dans l'image ci dessous) selon l'environnement dont vous allez créer votre travail.

Utiliser la vue rapport​

Lorsque vous chargez des données pour la première fois dans Power BI Desktop, vous verrez la vue Rapport avec le gabarit Power BI Desktop préchargé et configuré. Ce dernier inclus aussi un fond d'écran, la position du titre et des slicer par défaut ainsi qu'un guide d'alignement, afin de créer votre rapport dans les normes du projet.

Vous pouvez basculer entre les vues Rapport , Données et Modèle en sélectionnant les icônes dans le volet de navigation de gauche :

Une fois que vous avez ajouté des données, vous pouvez ajouter des champs à une nouvelle visualisation dans le canevas.

Dans la section Champs vous trouverez des tables de fait des mesures et des dimensions.

  1. les mesures sont créées et affichées dans la vue du rapport . Les mesures que vous créez vous-même apparaissent dans la liste Champs avec une icône de calculatrice. Vous pouvez nommer les mesures comme vous le souhaitez et les ajouter à une visualisation nouvelle ou existante comme n'importe quel autre champ.
  2. Les données que vous avez déjà chargées dans le modèle contient différents champs et des colonnes calculées qui apparaissent dans la liste Champs comme n'importe quel autre champ, mais elles auront une icône spéciale indiquant que leurs valeurs sont le résultat d'une formule (L'image au dessus contient la somme Classement comme colonne calculée lors de l'importation des données). Vous pouvez nommer vos colonnes comme vous le souhaitez et les ajouter à une visualisation de rapport comme les autres champs.

Créer des visuels​

Une fois le modèle de données élaboré, vous pouvez faire glisser des champs vers le canevas de rapport pour créer des visuels. Un visuel est une représentation graphique des données du modèle. Power BI Desktop propose de nombreux types de visuels. Le suivant représente un histogramme simple.

Il existe de nombreux types visuels différents disponibles directement à partir du volet Visualisations Power BI.

Pour créer ou modifier un visuel :

  • Dans le volet Visualisations, sélectionnez l’icône de visuel.

    Si un visuel est déjà sélectionné sur le canevas de rapport, il devient du type sélectionné.

    Dans le cas contraire, un nouveau visuel est créé en fonction de la sélection.

Types de visualisation dans Power BI​

Table​

Une table est une grille qui contient des données associées dans une série logique de lignes et de colonnes. Il peut également contenir des en-têtes et une ligne pour les totaux.

Les tableaux sont un excellent choix :

  • Pour voir et comparer des données détaillées et des valeurs exactes (au lieu de représentations visuelles).
  • Pour afficher les données sous forme de tableau.
  • Pour afficher des données numériques par catégories.

Matrice​

Le visuel matriciel est similaire à un tableau. Un tableau prend en charge deux dimensions et les données sont plates, ce qui signifie que les valeurs en double sont affichées et non agrégées. Une matrice facilite l'affichage des données de manière significative sur plusieurs dimensions .

La matrice agrège automatiquement les données et vous permet d'effectuer une analyse descendante.

Carte​

Parfois, un seul nombre est la chose la plus importante que vous souhaitez suivre dans votre tableau de bord ou rapport Power BI, comme le total des ventes, la part de marché d'une année à l'autre ou le nombre total d'opportunités. Ce type de visualisation s'appelle une carte . Comme avec presque toutes les visualisations natives de Power BI.

Segment(Slicer)​

Les segments dans Power BI sont un type de filtres visuels sur le canevas. Les segments permettent à un utilisateur de trier et de filtrer un rapport compressé et d'afficher uniquement les informations qu'il souhaite. Contrairement aux filtres, les segments sont présents sous forme de visuel sur le rapport et permettent à l'utilisateur de sélectionner des valeurs lors de l'analyse du rapport. Par exemple, si vous avez un rapport d'analyse des ventes, vous pouvez créer un slicer pendant des années. À partir de cette trancheuse, vous pouvez sélectionner l'année pour laquelle vous souhaitez voir les métriques de vente. Les visuels du rapport changeront automatiquement en affichant les informations pour cette année.

Vous pouvez utiliser un slicer dans votre Power BI lorsque vous souhaitez effectuer une ou plusieurs des choses mentionnées ci-dessous.

  • Pour accéder facilement aux filtres importants et fréquemment utilisés en les plaçant sur le canevas du rapport.

  • Affichez un rapport ciblé en plaçant le segment juste à côté des graphiques et des diagrammes essentiels.

  • Affichez facilement tous les filtres actuellement appliqués sur le rapport au lieu d'aller à la liste déroulante des filtres.

  • Pour afficher les colonnes inutiles/cachées/moins utilisées mais les garder pour les utilisateurs au cas où ils auraient besoin de les utiliser pour l'analyse.

Graphique combiné​

Dans Power BI, un graphique combiné est une visualisation unique qui combine un graphique en courbes et un graphique en colonnes. La combinaison des 2 graphiques en un seul vous permet de comparer plus rapidement les données.

Les graphiques combinés sont un excellent choix :

  • lorsque vous avez un graphique en courbes et un graphique en colonnes avec le même axe X.

  • pour comparer plusieurs mesures avec différentes plages de valeurs.

  • pour illustrer la corrélation entre deux mesures dans une visualisation.

  • pour vérifier si une mesure atteint l'objectif défini par une autre mesure

  • pour économiser l'espace de la toile.

Graphique en courbe​

Un graphique en courbes est une série de points de données représentés par des points et connectés par des lignes droites. Un graphique en courbes peut avoir une ou plusieurs lignes. Les graphiques en courbes comportent un axe X et un axe Y.

Vous pouvez convertir le visuel en graphique en courbes en sélectionnant le modèle Graphique en courbes dans le volet Visualisations.

Comment gérer et formater vos visuels?​

  1. Gérer les champs dans une visualisation: Les compartiments de ce volet varient en fonction du type de visualisation que vous avez sélectionné. Par exemple, si vous avez sélectionné un graphique à barres comme l'image au dessus, vous voyez Axe, Légende et Valeurs. Lorsque vous sélectionnez un champ ou que vous le faites glisser sur le canevas, Power BI ajoute ce champ à l'un des compartiments.

    Vous pouvez également faire glisser des champs de la liste Champs directement dans les compartiments. Par exemple, Values n'acceptera pas les champs non numériques. Ainsi, si vous faites glisser un champ Category dans le compartiment Values , Power BI le remplace par Count of Category .

  2. Formatez vos visuels: Vous devez sélectionner l'icône du rouleau de peinture pour afficher le volet Format. Les options disponibles dépendent du type de visualisation sélectionné.

Copier et coller entre les rapports​

Vous pouvez facilement prendre un visuel d'un rapport Power BI Desktop et le coller dans un autre rapport. Utilisez simplement le raccourci clavier Ctrl+C pour copier le visuel de votre rapport. Dans l'autre rapport Power BI Desktop, utilisez Ctrl+V pour coller le visuel dans l'autre rapport. Vous pouvez sélectionner un visuel à la fois, ou tous les visuels d'une page à copier, puis coller dans le rapport Power BI Desktop de destination.

Masquer les pages de rapport​

Lorsque vous créez un rapport, vous pouvez également masquer des pages d'un rapport. Cette approche peut être utile si vous devez créer des données ou des visuels sous-jacents dans un rapport, mais que vous ne souhaitez pas que ces pages soient visibles par d'autres, par exemple lorsque vous créez des tableaux ou des visuels de support qui sont utilisés dans d'autres pages de rapport.

Il est facile de masquer une page de rapport. Cliquez simplement avec le bouton droit sur l'onglet de la page du rapport et sélectionnez Masquer dans le menu qui apparaît.

Il y a quelques considérations à garder à l'esprit lorsque vous masquez une page de rapport :

  • Vous pouvez toujours voir une vue de rapport masquée dans Power BI Desktop, même si la page affiche une icône de page masquée. Dans l'image suivante, la page Présentation est masquée.

  • Vous ne pouvez pas voir une page de rapport masquée lors de l'affichage du rapport dans le service Power BI. Cependant , si vous regardez la page masquée lorsque vous enregistrez le rapport et le publiez sur le service Power BI, cette page sera la première que vos lecteurs de rapport verront.

  • Masquer une page de rapport n'est pas une mesure de sécurité. La page est toujours accessible aux utilisateurs et son contenu est toujours accessible à l'aide de l'extraction et d'autres méthodes.

Filtres​

Les niveaux de filtres dans Power BI​

Il existe trois niveaux de filtres dans Power BI : rapport , page et visuel .

  1. Les filtres au niveau visuel filtrent uniquement les données sur un visuel donné, qu'il s'agisse d'un tableau, d'un graphique, d'une carte, d'un segment, ou autres. Ce sont les filtres les plus granulaires que vous pouvez appliquer à vos données, et ils fonctionnent dans le contexte de la page. les filtres de niveau et de rapport, ce qui signifie que les filtres de niveau visuel ne peuvent pas les remplacer, ni être programmés pour filtrer les données sur d'autres visuels.
  2. Les filtres au niveau de la page filtrent uniquement les données sur une page donnée, ce qui les rend utiles pour créer des pages qui se concentrent sur des sous-ensembles particuliers de vos données. Par exemple, vous pouvez utiliser des filtres au niveau de la page pour qu'une page se concentre uniquement sur les données de revenus, tandis que la page suivante se concentre sur les données de dépenses. Les filtres au niveau de la page fonctionnent dans le contexte des filtres au niveau du rapport, ce qui signifie qu'un filtre au niveau de la page ne peut pas remplacer un filtre au niveau du rapport. Ils ne peuvent pas non plus être programmés pour filtrer les données sur d'autres pages.
  3. Les filtres au niveau du rapport sont ceux qui affectent toutes les données du rapport, indépendamment de ce que vous regardez. Considérez-les comme des filtres universels.

Masquer le volet Filtres en mode Lecture​

Si vous ne voulez pas du tout que les lecteurs de vos rapports voient le volet Filtres, sélectionnez l' icône en forme d' œil en regard de Filtres .

Masquer le volet Filtres lors de l'édition​

Vous pouvez masquer le volet Filtres pendant que vous modifiez votre rapport, lorsque vous n'utilisez pas le volet Filtres et que vous avez besoin d'espace supplémentaire à l'écran.

  • Dans l' onglet Affichage , le bouton Filtres vous permet d'afficher ou de masquer le volet Filtres.

Ce paramètre masque uniquement le volet Filtres dans Power BI Desktop. Il n'y a pas d'équivalent en mode édition dans le service Power BI.

Verrouiller ou masquer les filtres​

Vous pouvez verrouiller ou masquer des cartes de filtre individuelles. Si vous verrouillez un filtre, les consommateurs de votre rapport peuvent le voir mais pas le modifier. Si vous le cachez, ils ne peuvent même pas le voir. Masquer les cartes de filtre est généralement utile si vous devez masquer les filtres de nettoyage de données qui excluent les valeurs NULL ou inattendues.

  • Dans le volet Filtres, sélectionnez ou désélectionnez les icônes Verrouiller le filtre ou Masquer le filtre dans une carte de filtre.

Lorsque vous activez et désactivez ces paramètres dans le volet Filtres, vous voyez les modifications reflétées dans le rapport. Les filtres masqués n'apparaissent pas dans la liste de filtres contextuelle pour un visuel.

Publication de rapport sur le Power BI Service​

Un fois que vous terminez le travail sur Power BI Desktop, vous publiez le rapport et le jeux de données sur Power BI Service.

  1. Dans Power BI Desktop, sous l’onglet Accueil, cliquez ou appuyez sur Publier.

    Si vous n’êtes pas déjà connecté, connectez-vous au service Power BI, entrez un compte, puis cliquez ou appuyez sur Se connecter.

  2. Choisissez la destination du rapport, puis cliquez ou appuyez sur Sélectionner. Nous vous recommandons de publier dans un espace de travail de groupe pour simplifier l’accès au rapport.

    Afin que le titre du rapport soit significatif selon l'environnement ou vous allez le publier on rajoute dans le titre un _(Palier) (int, acc, prd) à la fin.

Après l'étape de publication de rapport ,le service Power BI charge le rapport dans un navigateur. Si le volet de navigation gauche n’est pas développé, cliquez ou appuyez sur le menu en haut à gauche (A) pour le développer.

Vous pouvez voir que, lorsque vous avez effectué la publication, Power BI Desktop a téléchargé un jeu de données (B) et un rapport (C).

Publier l'application Power BI​

Dans Power BI, vous pouvez créer du contenu packagé officiel, puis le distribuer à un large public en tant qu'application .

Lorsque les tableaux de bord et les rapports de votre espace de travail sont prêts, vous choisissez les tableaux de bord et les rapports que vous souhaitez publier, puis publiez-les en tant qu'application.

  1. Dans la vue de liste de l'espace de travail, décidez quels tableaux de bord et rapports vous souhaitez inclure dans l'application .
  2. Sélectionnez le bouton Créer une application en haut à droite pour lancer le processus de création et de publication d'une application à partir de l'espace de travail.

Une fois le rapport publier dans le service Cloud Power BI Service,il faut publier l'application et ajuster la sécurité afin de rendre disponible aux utilisateurs finaux le / les rapports.

  1. Dans Installation , saisissez le nom et la description pour aider les utilisateurs à trouver l'application. Vous pouvez également définir une couleur de thème.

  2. Sur Navigation , vous sélectionnez le contenu à afficher dans l'application. Ensuite, vous ajoutez la navigation dans l'application pour organiser le contenu en sections. Sous Avancé , vous pouvez définir la largeur par défaut du volet de navigation de gauche.

  3. Dans Autorisations , vous décidez qui a accès à l'application et ce qu'il peut en faire, et vous cochez Installation automatique pour que l'application s'installe automatiquement pour ceux qui ont l'autorisation.

  4. Lorsque vous sélectionnez Publier l'application , vous voyez un message confirmant qu'elle est prête à être publiée. Sélectionnez Publier .

  1. Dans la boîte de dialogue Publication réussie , vous pouvez copier l'URL qui est un lien direct vers cette application,ou vous pouvez envoyer ce lien direct aux personnes avec lesquelles vous l'avez partagé.

Après avoir publié votre application, vous souhaiterez peut-être la modifier ou la mettre à jour. Il est facile de le mettre à jour si vous êtes administrateur ou membre du nouvel espace de travail.

Revenez à la liste des contenus de l'espace de travail et sélectionnez Mettre à jour l'application dans le coin supérieur droit.

Mettre à jour la configuration , la navigation et les autorisations , si nécessaire, puis sélectionnez Mettre à jour l'application .

Les personnes à qui vous avez publié l'application voient automatiquement la version mise à jour de l'application.

Format_Dax_Calcul-DAX

Calcul DAX

Pourquoi DAX​

DAX = Data Analysis Expressions

C'est le langage qui permet d'exploiter les capacités analytique de Power BI. Cela représente 95 % du potentiel de Power BI. C'est comme dans Excel, Si on l'utilise sans connaitre et comprendre les formules Excel, la grille est en soit un outil plutôt limité. C'est la même chose avec Power BI. Comprendre et connaître les formules DAX donne accès à l'énorme potentiel de l'application.

Par exemple : Pour créer un graphique du nombre de vente par client il est possible de:

  1. Choisir un type de graphique : Graphique à barre groupé
  2. Sélectionner le champ Customer Name pour l'axe
  3. Sélectionner Quantity pour la valeur

Nous ne recommandons pas cette méthode. Il est même déconseillé d'utiliser les champs ou attributs quantitatifs comme valeur dans les visuels.

Il est très important de débuter toutes les analyses avec des mesures DAX. La raison est que même une simple mesure comme l'exemple précédent est la base pour passer au niveau suivant de l'analyse dans Power BI.

Pour créer le graphique précédent, en utilisant une mesure DAX, il faut d'abords créer la mesure.

  1. Sélectionner la table dans laquelle créer la mesures
  2. Dans l'onglet Outils de table, sélectionner l'option Nouvelle mesure

  1. Saisir la formule DAX suivante : 

    Quantité vendu = SUM(Sales[Quantity])

  2. La mesure apparaît dans la table sélectionnée avec une icone de calculatrice pour indiquer qu'il s'agit d'une mesure.

On peut maintenant recréer le même graphique en utilisant la mesure :

  1. Choisir un type de graphique : Graphique à barre groupé
  2. Sélectionner le champ Customer Name pour l'axe
  3. Sélectionner la nouvelle mesure Quantité vendu pour la valeur
  4. Le graphique créé est identique au précédent

L'exercice de créer les mesures de bases peut sembler superflu en premier lieu, mais elle comporte plusieurs avantages :

  • Le nom de la mesure n'aura pas à être modifié dans chacun des visuels où elle sera utilisée ;
  • Le nom de la mesure sera constant d'un visuel à l'autre, parce que non-modifié manuellement à chaque fois ;
  • Le formatage de l'unité de mesure est réalisé une seule fois (décimal, entier, pourcentage, etc.) ;
  • Si vous voulez utiliser n'importe quelle fonction plus avancée de DAX, comme les calculs de time intelligence, vous devez avoir créé les formules de base ;
  • L'ajout de règle de calcul, comme la quantité vendu, mais seulement dans le dernier mois ou pour un produit spécifique, lorsque fait avec une mesure, permet de ne pas refaire tous les filtres ou calculs pour arriver au résultat. La ou les règles de calcul sont intégrés dans la mesures.

En DAX, une formule est nommée mesure.

Il est important de donner un nom significatif et intuitif à chaque mesure. Cela améliore grandement l'utilisation et la compréhension du jeu de données.

Groupe de mesures​

Un groupe de mesures est en fait une table dans laquelle on regroupe plusieurs mesures.
On utilise les groupes de mesures pour organiser les mesures en sujet ou contexte de mesures.

Créer un groupe de mesure​

  1. Dans l'onglet Accueil

  2. Sélectionner l'option Entrer des données

  3. Saisir le nom du groupe de mesures, exemple : Mesure vente

  4. Charger la nouvelle table dans le jeu de données Power BI

  5. La table créée apparaît dans la liste des champs

Déplacer la ou les mesures déjà créées dans cette nouvelle table​
  1. Sélectionner la mesures à déplacer
  2. Dans l'onglet Outils de mesure, cliquer sur la liste déroulante de Table principale
  3. Sélectionner le nom de la table dans laquelle déplacer la mesure sélectionnée : Mesure vente

Supprimer le champ inutile​

Par défaut, lors de la création d'une table de saisie de données dans Power BI, un champ est toujours créé. Dans le cas des groupes de mesures, ce champ n'est pas utilisé et peut être supprimé.

  1. Dans le groupe de mesure, faire un clique droit sur le champ à supprimer
  2. Sélectionner supprimer du modèle dans le menu déroulant

Finaliser le groupe de mesures​
  1. Dans le panneau des Champs, faire un clique gauche pour fermer le panneau
  2. Un fois fermé, refaire un clique gauche pour ouvrir le panneau.
  3. La table créée apparaît en haut de la liste des tables avec une icone de calculatrice, c'est devenu un groupe de mesures.

Groupe de mesure dans le modèle de données​

  1. Sélectionner la vue Modèle

  2. Les groupes de mesures apparaissent dans la vue modèle comme un table indépendante.

    Les tables indépendantes ne sont pas liées par une relation aux autres tables.

    On recommande de placer à droite du modèle tous les groupes de mesures.

Organiser l'éditeur de requêtes​

En organisant les mesures, puisque l'on crée toujours plusieurs groupe de mesures, il est important de conserver le modèle organisé.

Dans l'éditeur de requête, on déplace le groupe de mesures dans un nouveau groupe de requêtes nommé groupe de mesure.

  1. Dans l'onglet Accueil, sélectionner l'option Transformer les données pour ouvrir l'éditeur de requêtes.

  1. Dans la section requête, faire un clique droit sur Mesure vente
  2. Dans le menu déroulant, sélectionner l'option Déplacer vers le groupe
  3. Choisir l'option Nouveau groupe...

  1. Dans la fenêtre Nouveau groupe, saisir le nom : Groupe de mesures et cliquer sur OK

  1. Cela crée un groupe de requêtes nommées Groupe de mesures qui permettra de regrouper tous les groupes de mesures créés.

    Cela permet ainsi de garder l'éditeur de requête organisé.

Mesure simple​

Par défaut, la mesure sera créée dans la table sélectionnée. Avant de créer une mesure, toujours sélectionner le groupe de mesures dans lequel la nouvelle mesure doit s'ajouter.

Créer une nouvelle mesure​
  1. Sélectionner le groupe de mesures dans lequel créer une nouvelle mesure

  1. Dans l'onglet accueil, sélectionner l'option Nouvelle mesure
  2. Saisir le nom de la mesure et la formule DAX associée.

Exemple de mesure simple :

Ces mesures appliqueront la fonction choisie sur l'ensemble de la colonne sélectionnée.

  • SUM() = Calcul la somme d'une colonne
  • AVERAGE() = Calcul la moyenne des valeurs d'une colonne
Quantité vendu = 
SUM(Sales[Quantity])

Quantité moyenne vendu = 
AVERAGE (Sales[Quantiy])


  • SUMX() = Fonction itérative ligne par ligne. Toutes les fonctions X appliquent un calcul sur chaque ligne avant d'appliquer la fonction sur la colonne.

  • Par exemple, pour calculer le montant de la vente de chaque transaction, on peut créer une colonne calculée qui multiplie la quantité par le prix de la vente. Cela donne le montant de la vente qu'on peut ensuite additionner pour avoir les ventes totales.

La fonction itérative SUMX() permet d'obtenir le même résultat sans créer de colonne calculée.

Il faut préciser la table sur laquelle itérer et ensuite le calcul à effectuer.

Vente totale = 
SUMX(Sales, Sales[quantiy] * Sales[Price])
  • COUNTROWS() = Calcul le nombre de lignes ou d'enregistrement dans une table
Nombre de ventes = 
COUNTROWS = (Sales)

On peut ensuite utiliser les mesures ensemble dans un même visuel.

Utiliser les informations d'une table liée dans les mesures​

Supposons que vous voulez calculer le profit réalisé sur les ventes. Cette information n'est pas présente dans le jeu de données. Par contre, nous disposons des informations nécessaires pour le calculer.

On a déjà fait une mesure pour calculer le montant des ventes.

La table produite contient la valeur du coût du produit. Si on multiplie le coût des produits par le nombre d'unités vendues, cela nous donnera le coût total des ventes.

Il est important de noter que le champ Cost n'est pas dans la table Sales, mais plutôt dans la table Product.

On a vue que les fonctions X, comme SUMX(), nous permettent d'itérer sur une table pour faire un calcul ligne par ligne avant d'appliquer la fonction sur toute la table. Par contre, dans notre cas, les deux champs à multiplier ne sont pas dans la même table.

  • Cost est dans la table Product
  • Quantity est dans la table Sales

L'image suivante montre que ces deux tables sont liées dans le modèle par le champ Product ID qui est présent dans les deux tables et permet ainsi de créer une relation.

On peut donc utiliser la fonction RELATED() pour rechercher un champ dans une table liée et l'utiliser dans les formules comme s'il était présent dans la table.

Coût total = 
SUMX( Sales, Sales[Quantity] * RELATED( Product[Cost] ) )

Mesure Branching on met quoi en francais​

Créer des mesures à partir d'autres mesures.

Power BI permet de réutiliser les mesures déjà créées dans d'autres mesures. Cela permet de simplifier et ne pas répéter le même code dans plusieurs mesures différentes.

Pour terminer le calculer du profit réalisé par les ventes, Il faut soustraire les coûts totaux des ventes total. On peut aussi calculer la marge de profit en divisant les bénéfices par les ventes totales.

Bénéfice = 
[Vente total] - [Coût total]

Marge de profit = 
DIVIDE([Bénéfice],[Vente total])

Les deux mesures précédentes utilisent le branching de mesure. C'est à dire que des mesures sont réutilisées dans d'autre mesures.

Lors du calcul du bénéfice, on a pas à se soucier du calcul des ventes totales ni du calcul du coût total. Ces derniers sont gérés dans les mesures créés précédemment et sont calculés automatiquement lors du calcul du bénéfice.

Cela ouvre d'énormes possibilités. Vous pouvez réutiliser ces mesures de bases pour calculer:

  • La valeur de la mesures à la même date ou période l'an passer ;
  • La valeur depuis le début de l'année ;
  • La valeur pour un mois ou un trimestre ;
  • Un moyenne mobile sur plusieurs mois ;
  • La valeur pour une catégorie de client spécifique ;
  • Et ceci est seulement quelques exemples.

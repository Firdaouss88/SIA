Les fonctions de table DAX

Introduction aux fonctions de table​

Les fonctions de table sont des fonctions DAX qui sert à manipuler et contrôler les tables. Les fonctions les plus importantes sont FILTER , ALL , VALUE , DISTINCT , RELATEDTABLE.

Filtrage d'une table​

La fonction FILTER est une fonction qui filtre une table et elle a deux arguments, le premier une expression de table , le deuxième est une expression qui sera exécuté pour chaque ligne de cette table. Commençons par le code suivant, la fonction SUMX() évalue le premier argument qui est FILTER , FILTER reçoit un nombre de lignes et évalue la condition Orders[Price] > 1 , donc seulement les lignes satisfaites à la condition de filtre seront retournées par FILTER. Ensuite l'itération de la fonction SUMX() se fait sur les lignes obtenues par FILTER, alors pour chaque ligne on calcule la quantité fois le prix .

SUMX(
    FILTER (
            Orders,
            Orders[Price] > 1
    ),
    Orders [Quantity] * Orders [Price]
)

Le comportement de la fonction FILTER produit techniquement un plus petit nombre de lignes en résultat et ces derniers doivent se faire itérer par une fonction 'X' ,comme le cas de SUMX() qui itère ligne par ligne toute la table retournée par le filtre.

La fonction FILTER obtient une table et renvoie une table en enlevant quelques rangées de la table initiale qu'elle a reçue.

Ignorer les filtres​

La fonction ALL renvoie tout le contenu d'une table, en ignorant les filtres qui existent sur cette table. Dans l'exemple suivant, nous utiliserons ALL comme argument de SUMX(), SUMX() va recevoir le résultat de ALL et pour chaque rangée elle va évaluer la quantité multipliée par le prix et enfin résumer les résultats. 

SUMX (
    ALL( Orders ),      
    Orders [Quantity] * Orders [Price]
)

Une fois qu'on applique cette mesure dans notre rapport , on obtiendra des valeurs identiques pour toutes les lignes sur le tableau

On peut servir de cette mesure en créant une mesure Pourcentage comme :

PCT = DIVIDE ( [Sales Amount], [ ALL Sales],0)

ALL a aussi une syntaxe avec une seule colonne ou plus comme: ALL(Customers [CusotmersName]) retourne une table calculé qui n'a qu'une seule colonne et cette colonne contient que les valeurs uniques de la colonne CustomersName.

Il existe aussi la fonction ALLEXCEPT, c'est une fonction particulière, comme le cas où vous avez besoin d'écrire ALL avec toutes les colonnes d'une table en excluant une ou deux colonnes,vous pouvez à la place écrire ALLEXCEPT avec le champs à exclure:

Combiner les filtres​

Commençons par le code suivant qui combine 3 fonctions SUMX(), ALL , FILTER :

SUMX(
    FILTER (
            ALL ( Orders) ,
            Orders[Price] > 1
    ),
    Orders [Quantity] * Orders [Price]
)

Donc l'exécution de cette mesure évalue SUMX(), et SUMX() tente d'évaluer le premier argument FILTER, et la fonction FILTER doit évaluer ALL (Orders) . Maintenant la première fonction de cette expression qui est évaluée et bien la fonction ALL, ALL reçoit les lignes de la table Orders et elle va ignorer le filtre comme on a déjà vu ALL ignore tout filtre appliqué sur la table ,le résultat de la fonction ALL dans ce cas est la table entière Orders. Le comportement de filtre est d'appliquer la condition Orders[Price] > 1 sur les lignes de la table, le résultat du filtre est juste les lignes qui respectent cette condition. En dernier, la fonction SUMX() va itérer les lignes qui acceptait la condition du filtre est évaluer la quantité multipliée par le prix pour chaque rangée.

Nous pouvons utiliser différentes fonctions de table et les combiner comme vous l'Avez vous pour FILTER et ALL.

Distinct​

La fonction DISTINCT renvoie les valeurs uniques d'une table, et elle respecte l'application des filtres existants dans un rapport.

L'exemple ci dessous va nous retourner le nombre de lignes des valeurs uniques du code de produit :

NumOfProducts = 
                COUNTROWS (
                            DISTINCT( Product [ ProductCode ])
                )

Nous avons aussi la fonction DISTINCTCOUNT pour calculer le nombre distincte de lignes .

ALLSELECTED​

La fonction ALLSELECTED est une fonction similaire à la fonction ALL , mais avec un comportement particulier qui s'impacte avec les filtres qui se trouvent dans le rapport :

SUMX(
            ALLSELECTED ( Orders) ,
            Orders [Quantity] * Orders [Price]
)

RELATEDTABLE​

RELATEDTABLE est une fonction qui est similaire à RELATED,mais elle commence du côté de relation (1) vers côté relation (*) et nous renvoie une table contenant toutes les lignes qui concernent l'élément du côté (1).

Donc selon l'exemple au dessus, un client peut faire plusieurs ventes et une vente va être faite qu'avec un seul client. En se basant sur ce modèle on peut calculer le montant de la transaction de ventes faites par un client , la mesure est la suivante:

SalesOfCurrentCustomer =
            SUMX( 
                    RELATEDTABLE (Sales), 
                    Sales [ Quantity] * Sales [ Net Price]
    )

Tables et relations​

Dans un modèle tabulaire, une relation physique relie deux tables via une équivalence sur une seule colonne. Les relations ont une cardinalité, dont il existe trois types. La différence entre eux est à la fois technique et sémantique. Voici les trois types de cardinalité de relation disponibles :

  • Relations un-à-plusieurs : il s'agit du type de cardinalité de relation le plus courant. D'un côté de la relation, la colonne doit avoir des valeurs uniques ; du côté multiple, la valeur peut (et contient généralement) des doublons. Tout dépend de l'ordre des tables : une relation un-à-plusieurs entre Produit et Ventes est identique à une relation plusieurs-à-un entre Ventes et Produit .
  • Relations un-à-un : il s'agit d'un type de cardinalité de relation plutôt rare. Des deux côtés de la relation, les colonnes doivent avoir des valeurs uniques. Un nom plus précis serait une relation « zéro ou un » à « zéro ou un », car la présence d'une ligne dans une table n'implique pas la présence d'une ligne correspondante dans l'autre table.
  • Relations plusieurs-à-plusieurs : des deux côtés de la relation, les colonnes peuvent avoir des doublons. En effet, dans le langage courant de modélisation de données, « plusieurs-à-plusieurs » fait référence à un autre type d'implémentation, créé en utilisant des paires de relations un-à-plusieurs et plusieurs-à-un.

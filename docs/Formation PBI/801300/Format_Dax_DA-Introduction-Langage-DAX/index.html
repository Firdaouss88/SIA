Introduction au langage DAX Avancé

Introduction DAX​

Dans cette formation, vous allez avoir une idée sur la syntaxe du langage DAX, la différence entre une colonne calculée et une mesure (également appelée Champ calculé, dans la terminologie Excel), et les fonctions les plus couramment utilisées dans DAX. Vous apprendrez également la syntaxe des variables , qui ont été introduits dans DAX pour Power BI Desktop, Excel 2016 et Analysis Services 2016.

C’est quoi DAX ?​

Le langage DAX (Data Analysis Expression) est un langage d’analyse de données très puissant. Il est utilisé par Power Pivot dans Excel, Power BI Desktop et SSAS (SQL Sever Analysis Services) tabulaire. Il s’agit d’un langage inspiré en partie des fonctions Excel mais adapté à la réalité d’analyse de bases de données. Il permet notamment la création d’indicateurs sur mesure répondant aux problématiques des utilisateurs métiers.

Les différents types de données​

Quel que soit le type de données que vous avez dans la source, quand vous chargez des données dans Power bi, elles sont converties dans l'ensemble des données types disponibles dans Dax . Il y a plusieurs types de données:

Type de donnéesDéscription
Integer (64 bits)Les entiers sont toujours stockés sous forme d'entier 64 bits.
Decimal (floating point)Nombre flottante avec n'importe quel nombre de décimales .
Currency (Money)la monnaie spécifiquement pour les devises vous avez le type de données de devise également connu sous le type de données d'argent gardez à l'esprit devises sont stockées sous forme d'entier .
Date (DateTime)Les données DateTime sont stockées dans Dax comme valeur à virgule flottante où la partie entière représente le nombre de jours après le 30 décembre 1899 , par conséquent 1en tant que date représente le 31 décembre 1899, 2 est le 1 janvier 1900 .
Vous pouvez donc en ajouter un à une date et vous Obtenez le lendemain au soustraire un d'un jour et vous obtenez la veille.

Quelle est la différence entre modifier le type de données d'une colonne et changer la visualisation de format d'une colonne ? :

  • Si vous modifiez le type de données d'une colonne vous changez la manière dont les données sont représentées et stockées dans le modèle .

  • Si vous changez la chaîne de format des données , elles seront toujours conservées mais vous allez les montrer à l'utilisateur final seulement avec format que vous avez choisi .

    Donc la différence est de savoir si vous voulez stocker des données d'une manière ou vous voulez juste les montrer d'une certaine format.

Les colonnes calculées​

Pour apprendre le langage, la première chose à apprendre et de savoir créer une colonne calculée. Une colonne calculée est fondamentalement un calcul que vous faites à l'intérieur de votre table, afin d'enrichir votre modèle de données. Exemple: DiffDate est la date de livraison moins la date de commande .

DiffDate = Sales[Delivery Date] - Sales[Order Date]

une fois vous appuyez sur Entrée le moteur calculera la valeur de l'expression pour chaque ligne de votre table et ajoutera le résultat à la fin. Exemple : Calculer l'âge d'un client avec une simple équation, on prend l'année' d'aujourd'hui moins l'année de la date de naissance du client. Âge= YEAR(TODAY()) - YEAR(Customer[Birth date]) Une fois vous appuyez sur Entrée vous aurez une colonne Âge qui est ajoutée à la fin de votre table et à chaque actualisation des données, le contenu sera recalculé en stockant dans la table des clients la colonne âge de client.

  • Lorsque vous écrivez une expression Dax qui fait référence à des valeurs dont le cas d'une colonne calculée, elle est toujours exécutée pour une ligne et le calcul se fait ligne par ligne donc les informations sur quelle ligne doit être utilisé est implicitement définie dans la colonne calculée .

  • Quand vous créez une colonne calculée , ce que vous faites ,c'est de changer votre modèle vous créez une nouvelle colonne qui est calculé dans Dax et elle est stockée dans le cadre de votre table donc vous ne travaillez pas au niveau du rapport vous ne résumez pas ceci valeur plus valeur mais ligne par ligne pour tous les lignes de votre table le calcul arrive et le résultat est stocké dans votre modèle.Cela signifie que les colonnes calculées utilisent de l'espace dans votre modèle .

  • Vous ne pouvez pas faire tout vos calculs avec des colonnes calculées, les colonnes calculées sont bien si vous faites des calculs lignes par ligne mais si vous avez besoin de faire des pourcentages alors ce qu'il faut faire est de créer des mesures et non des colonnes calculées.

La façon dont nous référençons les colonnes sont toujours à utiliser le nom de la table puis le nom de la colonne entre crochets. La syntaxe correcte serait le nom de la table entre guillemets simples puis le nom de la colonne entre crochets 'TableName'[ColumnName]. Vous pouvez éviter d'utiliser les guillemets simples si votre nom de table fait ne contient pas d'espace ou ne commence pas par un nombre .

Mesures​

Une mesure et une expression Dax qui au lieu d'être calculé ligne par ligne puis stocké dans la table il est calculé au niveau du rapport. Il y'a une différence entre colonnes calculées et les mesures, une colonne calculée est calculée rangée par rangée et stockée dans le modèle. Les mesures en revanche sont calculées au moment de la requête, elles agrègent les valeurs et ensuite elles calculent le résultat.

Colonne Calculée : Line Margin = Sales[Quantity]  * (Sales[Net Price] - Sales[Unit Cost])
Masure : Margin = SUM ( Sales[Line Margin] )
Masure : MarginPerc = [Margin] / SUM(Sales[Line Amount])

Ici la mesure Margin% va faire agrégation de Line Margin divisée sur agrégation de Line Amount. En termes de Processeur et d'espace,les colonnes calculées utilisent de l'espace à la fois sur le disque et la RAM alors que les mesures utilisent uniquement le processeur, donc pour les bonnes pratiques on évite les colonnes calculées et on essaie de tout calculer en utilisant des mesures.

En tant que convention de nommage les mesures n'ont jamais de nom table devant eux , la raison est que nous devons être capables de distinguer entre une colonne et une mesure .Dans le code la norme que tout le monde utilise est de ne jamais mettre le nom de la table devant mesure et toujours mettre le nom de la table devant les colonnes . Vous devez essentiellement utiliser une colonne quand vous voulez être capable de trancher par quelque chose. Par exemple, si vous voulez classer les prix si vous voulez calculer l'âge d'un client ,dans ces cas une colonne calculée est nécessaire. Dans d'autres scénarios une Mesure est votre meilleur option.

Les fonctions d'agrégation​

Les fonctions d'agrégation sont utiles à chaque fois que vous souhaitez agréger des valeurs. Les fonctions les plus couramment utiliées sont SUM (Somme), AVERAGE(Moyenne), MIN (Minimum), MAX (Maximum) Exemple : Sales Amount = SUM(Sales[Line Amount]) Les fonctions d'agrégation ont deux limitations, la première c'est qu'elles fonctionnent que sur les chiffres et la deuxième c'est qu'elles prennent une seule valeur (une seule colonne à calculer). Pour cette raison , il existe des fonctions d'agrégations 'X' qui sont SUM X(),AVERAGE X(),MIN X(),MAX X().... qui s'appellent aussi des itérateurs. La nature de ces itérateurs,au lieu de calculer uniquement un agrégat,la fonction va scanner une table et itérer ligne par ligne et calculer l'expression pour chaque ligne,par exemple: SUMX ( Sales, Sales [Price ] * Sales [Quantity] ). Nous fournissons comme première paramètre la table et le deuxième paramètre l'expression qui peut calculer les valeurs ligne par ligne de la table Sales.

Disons par exemple au lieu de découper par catégorie je veux analyser mes produits en les séparant en 2 grandes catégories je veux avoir un produit pas cher et produit cher , un produit bon marché est tout produit dont le prix était inférieur à 100$ et un produit cher a un prix plus élevé et je vais pouvoir trancher et voir le montant des ventes de produits bon marché contre les cher ,dans ce cas j'ai besoin d'utiliser une colonne parce que je veux calculer une chaîne qui sera utilisée pour trancher . Price Level= if (‘Product’ [Unit Price] > 100, ‘’EXPENCIVE’’, ‘’ CHEAP ‘’) Et maintenant je peux utiliser cette colonne Price Level pour trancher et séparer les produits dont ces 2 catégories. La colonne ayant une petite cardinalité est une très petite colonne et ne crée aucun problème .

Les valeurs de comptage (Counting values)​

Il y a beaucoup d'agrégateurs ,pas seulement SUM et MIN. Par exemple, nous avons des agrégateurs qui compte des valeurs ,nous avons COUNT / COUNTA ils comptent n'importe quoi mais il ne compte pas les blancs et nous avons COUNTBLANK qui ne compte que des blancs et puis nous avons les agrégateurs les plus largement utilisés qui sont COUNTROWS (compte le nombre de lignes dans une table) et DISTINCTCOUNT (compte le nombre des valeurs distinctes d’une colonne ).

Dans le cas où nous avons une colonne par exemple CustomerKey qui identifie le client qui a acheté un produit dans la table Ventes donc en calculant le nombre des ventes distinctes par client nous pouvons projeter cela comme: # Customers = DISTINCTCOUNT ( sales [CustomerKey] ). Ce type de calcul est déjà utile pour lui-même mais il pourrait aussi être utile pour calculer par exemple le montant moyen par transaction ou le montant moyen par client : Sales by Customer = [Sales Amount] / [ # Customers]

Les fonctions conditionnelles​

il existe un ensemble de fonctions dans Dax qui sont utiles pour calculer la logique booléenne ,nous avons déjà utilisé IF , mais IF n'est qu'une partie de ses fonctions ,les fonctions sont AND, OR et NOT. IF est une fonction pour vérifier une condition et ensuite fournir des résultats différents. AND, OR et NOT sont des fonctions booléennes standard. Donc vous pouvez avoir 2 conditions que vous pouvez mettez dans un AND logique dans un OR logique ou utiliser une négation booléenne.

vous avez 2 façons d'écrire AND et OR : AND ( A, B ) = A && B OR ( A, B )= A || B

une fonction n'est utile que pour vérifier 2 conditions à la fois, par contre les opérateurs supportent plusieurs conditions è la fois comme :

Color Type =
IF ( 
     ‘Product’[Color]= ‘’Black’’ || ‘Product’[Color]= ‘’Pink’’  || ‘Product’[Color]= ‘’White’’   ),
    ‘’Trendy’’,
    ‘’Boring’’
)
Color Type =
IF ( 
     ‘Product’[Color] IN {‘’Black’’ , ‘’Pink’’  , ‘’White’’},
    ‘’Trendy’’,
    ‘’Boring’’
)

Il y a une autre fonction qui est un peu liée aux fonctions conditionnelles qui est la fonction SWITCH. SWITCH est différent des autres fonctions, elle évalue une expression , donc le premier argument est une expression la couleur du produit et puis la prochaine série des arguments à venir est sous forme de : le premier est la valeur de l'expression et le 2ème est la valeur du résultat .

Color Type =
SWITCH (
     ‘Product’[Size] ,
     ‘’ S ’’ , ‘’ Small ’’,
      ‘’ M ’’  , ‘’ Medium’’, 
      ‘’ L ’’ ,‘’ Large ’’,
        ‘’ XL ’’ ,’’ Extra Large’’,
        ‘’ Other ’’
   )

Il y a une manière plus créative d'utiliser la fonction SWITCH, c'est quand tu veux écrire des expressions complexes. Vous pouvez activer une valeur constante ,dans ce cas nous basculons sur TRUE () le premier argument de SWITCH est donc toujours True et ensuite vous fournissez un ensemble de conditions .

DiscountPct  =
SWITCH(
      TRUE (),
      Product [Size] = ‘’S ‘’ , 0.5 ,
      AND ( Product [Size] = ‘’L ‘’ , Product [Price] < 100 ) , 0.2 ,
     Product’[Size] = ‘’ L ‘’ , 0.35 , 
     0
 )

La première condition qui sera identique à True , cela signifie que la première condition qui vaut True produira le résultat. Donc si la taille du produit est égal à S le résultat est 0.5 si la taille du produit est égale à L et le prix est inférieur à 100 alors le résultat et 0.2 sinon si la taille est égale à L le résultat est 0.35 , sinon 0.

Gestion des erreurs​

Votre code d'accès peut contenir des erreurs .Il pourrait y avoir des erreurs de syntaxe comme dans le cas où vous pouvez écrire une expression comme celle-ci : SUMX ( Sales, Sales [Quantity] ) et la quantité vendue pourrait aussi bien être une chaîne et le moteur va essayer de faire la conversion ligne par ligne de la chaîne en un nombre si à un moment donné ils échouent dans la transformation cela produira une erreur et vous ne voulez pas que cette erreur arrive à l'utilisateur final . Il est généralement préférable d' intercepter ce genre d'erreur et il y a une fonction de traitement ISERROR dans Dax , elle essaie de calculer une expression et selon que l'expression produit une erreur , ou non , renvoie vrai ou faux . Il existe une autre fonction IFERROR , cette fonction tente de calculer une expression en cas d'erreur alors il retourne le 2e argument qui sera la valeur par défaut ou la valeur alternative .

IFERROR( 
    Sales [ GrossMargin] / Sales [ Amount ] ,
    BLANCK ()
)
DIVIDE ( 
    Sales [ GrossMargin] ,
     Sales [ Amount ] ,
    0
)

Dans le cas de la division, pour éviter toute erreur, on utilise la fonction DIVIDE si vous lui fournissez que deux arguments la 3ème valeur par défaut est un blanc généralement vide et la valeur que vous voulez revenir au cas ou vous faites une division par 0.

Utilisation des variables​

Une fonctionnalité extrêmement utile de Dax et celle des variables vous pouvez introduire variables dans vos formules afin de raccourcir votre code et le rendre plus lisible. La syntaxe est assez simple vous introduisez une variable avec un mot-clé VAR et vous devez également fournir la partie RETURN dans la partie Return vous pouvez utiliser les variables définies dans la partie Var et à chaque fois que vous écrivez le nom de la variable vous voulez dire la valeur de l'expression stocké dans la variable . Les variables sont utiles pour beaucoup de raisons les 2 plus convaincants et là lisibilité et la performance . 

VAR 
    TotalQuantity= SUM( Sales [Quantity] )
RETURN
    IF (
        TotalQuantity > 1000 ,
        TotalQuantity * 0,95 ,
        TotalQuantity * 1,25
)

Dans cet exemple la variable TotalQuantity calcule la somme de la quantité vendue et dans la partie Return, vous calculez si la quantité totale est supérieur à 1000 alors vous faites la quantité totale multiplié par 0,95 sinon vous la multipliez à 1,25.

Les fonctions mathématiques​

Dax offre également un riche ensemble de fonctions qui fournissent des maths,arrondi, calcul de texte,calcul de date.

  • Les fonctions mathématiques comprennent la valeur absolue ABS affacturage FACT logarithme LOG les cautions de puissance QUOTIENT et d'autres fonctions qui font des calculs simples, ces derniers travaillent de la même manière comme ils le font dans Excel.
  • Les fonctions d'arrondi font l'arrondi de la valeur donné en paramètre et elles retournent toujours une valeur décimale: FLOOR (Value, 0.01) ROUNDDOWN ( Value ,2 ) MROUND ( Value , 0.01) CEILING ( Value , 0.01)
  • Casting Fonction, ces fonctions changent quel que soit le type de données que vous leur passez soit à un entier ou une monnaie, n'oubliez jamais que l'arrondissement n'est pas la même chose que le casting . INT (Value)CURRENCY (Value)
  • Les fonctions de texte sont très similaires à leurs contreparties dans Excel vous pouvez concaténer des chaînes vous pouvez trouver les occurrences d'une sous-chaîne en une chaîne vous pouvez les changer pour minuscule a majuscule comme : CONCATENATE ,FIND , FIXED ,LEFT ,LOWER ,LEN ,REPLACE ,UPPER ,SUBTITUTE…
  • Les fonctions de date ce sont des fonctions qui prennent des dates comme paramètre et ils peuvent extraire la fin d'un mois d'une date ou l'année d'une date ou vous renvoyez la date d'aujourd'hui ils font des calculs en plus sur des dates comme : DATE,DATEVALUE,EOMONTH,HOUR,NOW,SECOND,TIMEVALUE,TODAY,WEEKDAY,YEAR,WEEKNUM…

Les fonctions relationnelles​

Il existe deux fonctions relationnelles RELATED et RELATEDTABLE elles sont utiles pour créer des chaines de relations entre deux tables. Dans chaque relation dans un modèle on a le côté qui contient 1 et le côté qui contient plusieurs ,d'un point graphique de vue Power bi montre 1 d'un côté et * sur le côté nombreux.

  • RELATED fonctionne du côté multiples et vous donne accès à colonne d'un côté (1),donc vous pouvez merger vos données et stocker les dans le cas de sous-catégories dans la table du produit en utilisant la fonction relationnelle RELATED .
Subcategory = RELATED ( 'Product Subcategory' [ Subcategory] )
Category = RELATED ( 'Product Category' [ Category ])

Quelle est la signification de One to Many ?
pour une catégorie il y a beaucoup de sous-catégories pour une sous-catégorie il y a beaucoup de produits mais étant donné un produit à une seule sous-catégorie et une seule catégorie .

  • RELATEDTABLE renvoie une table contenant toutes les lignes d'une table du côté multiples qui ont une relation avec la ligne en cours sur la table sur laquelle on définit la colonne où nous écrivons l'expression. 
Number of sales = COUNTROWS ( RELATEDTABLE  ( Sales ) )

RELATEDTABLE ne renvoie que les lignes dans les ventes qui ont une relation avec la ligne en cours donc dans cet exemple ci dessus ,le résultat contient toutes le nombre de ventes de la catégorie donnée.

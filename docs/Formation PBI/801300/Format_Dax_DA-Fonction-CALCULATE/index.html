Format_Dax_DA-Fonction-CALCULATE

Fonction CALCULATE

Introduction​

CALCULATE est de loin la fonction la plus importante, la plus utile et la plus complexe du langage DAX. La fonction elle-même est simple. Il n'exécute que quelques tâches, mais le nombre de scénarios où CALCULATE est nécessaire, ainsi que la complexité des formules qui peuvent être écrites avec CALCULATE, obligent à lui consacrer un document complet.

À l'aide de CALCULATE, vous pouvez modifier le contexte de filtre qui évalue une expression DAX. Vous pouvez modifier les filtres au niveau de la granularité de la colonne et vous pouvez remplacer ou combiner des filtres existants avec de nouveaux filtres.

Vous pourrez appliquer les mêmes concepts à CALCULATETABLE, qui ressemble à CALCULATE mais renvoie une table au lieu d'une valeur unique.

Exemples CALCULATE​

La fonction CALCULATE peut manipuler le contexte du filtre en ajoutant ou éliminant le filtre, on commence par l'exemple CALCULATE avec ALL :

SalesAllColors = 
        CALCULATE (
                        [Sales Amount],
                        ALL ( Product[ Color])
)

ALL est un modificateur de filtre, lorsqu'elle est utilisée avec la fonction CALCULATE comme fonction unique, elle va supprimer le filtre appliqué sur Product [Color] s'il y'en a. Donc on va avoir comme résultat le montant des ventes pas tout couleur de produit , malgré s'il existait un filtre sur la colonne couleur.

Nous avons aussi des combinaisons composé à base de la fonction CALCULATE et la fonction KEEPFILTERS, cette dernière fonctionne uniquement en tant qu'argument filtre de la fonction CALCULATE:

SalesTrendyColors = 
        CALCULATE (
                        [Sales Amount],
                        KEEPFILTERS ( Product[ Color] in { "Red" , ''Blue"})
)

Les opérateurs internes pour le contexte du filtre​

La fonction CALCULATE peut modifier le contexte de filtre de différentes manières, lorsque vous rajouter la fonction ALL, vous supprimez tout filtre existant sur la colonne ou la table passée en argument. En utilisant la fonction KEEPFILTERS vous ajoutez un nouveau filtre dans le contexte du filtre sans enlever les filtres existants en avant. Les opérations appelées interne sont celles qu'on applique en interne du contexte de filtre et qu'on peut les utiliser avec la fonction CALCULATE, comme la fonction INTERSECT, REMOVEFILTERS, ADDFILTER , OVERWRITE.

Utiliser une seule colonne dans la syntaxe compacte​

Commençons par le code suivant :

BigSales = 
        CALCULATE (
                        [Sales Amount],
                        FILTER (
                                Sales,
                                Sales[Quantity] * Sales [Net Price] > 1000

)

Dans ce cas la fonction FILTER va retourner une table qui va contenir la liste des combinaisons uniques de la quantité multiplié par le prix net soit supérieur à 1000 pour toute la table Sales.

Vous pouvez optimiser plus votre mesure par le fait de garder que les colonnes essentielles pour vos calculs,en utilisant par exemple la fonction ALL avec les deux colonnes quantité et prix net :

BigSales 2 = 
        CALCULATE (
                        [Sales Amount],
                        FILTER (
                                ALL ( Sales[Quantity],Sales [Net Price])
                                Sales[Quantity] * Sales [Net Price] > 1000
)

Le résultat de la fonction ALL est une table avec deux colonnes et ne contient plus toutes les colonnes de la table ventes, donc vous obtenez une table avec moins de lignes parce que la sélection est unique des colonnes quantité et prix net de la tables des ventes, mais la différence ici que ALL élimine tout filtre appliqué sur la table ventes. Donc on filtre sur les deux colonnes sans aucun filtre sur leur table source et avec la sélection unique des champs( pas de doublon).

Aussi une autre remarque , il ne faut pas utiliser l'agrégation dans une syntaxe compacte comme suit:

CALCULATE (
                        [Sales Amount],
                        Sales[Quantity] < SUM (Sales [Quantity] ) / 100
)

Ce calcul va essayer de filtrer uniquement les transactions où la quantité est moins d'un pourcent de la somme de la quantité de toute la table des ventes, alors qu'il est déconseillé d'inclure l'agrégation dance ce contexte.

Ce qui est efficace est de déplacer l'évaluation de l'agrégation avant le calcul dans une variable où vous allez enregistrer la valeur calculée et après utiliser cette variable dans vos filtres:

VAR TotalQuantity =  SUM (Sales [Quantity] )
RETURN
CALCULATE (
                        [Sales Amount],
                        Sales[Quantity] < TotalQuantity  / 100
)

Cela rend le code plus facile à lire et plus facile à exécuter.

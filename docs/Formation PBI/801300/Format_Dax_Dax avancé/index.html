  1. Comprendre le Contexte, c'est très important
  2. Commencer avec des formules simples

  4. La fonction la plus utilise est Calculate, la comprendre apporte une pouvoir analytique immense
  5. Pratiquer... pratiquer... pratiquer

Format de données​

Au chargement des données, on voit le type de données déterminé par Power BI dans l'éditeur de requête (Transform data)

  • Valider que les attributs numérique et de date on le bon format (entier, décimal, monétaire, date, time etc)

On voit le type de données à gauche du libellé de chacune des colonnes

On peut changer le type de données d'une colonne via:

  • Cliquer directement sur le type à droite du libellé

  • Dans le menu ruban, dans l'onglets Home, sous Data type:

  • Sous l'option de grille dans Power BI Desktop, dans l'onglet Column option

Syntaxe des formules DAX​

Vous devez penser en table et colonne

Un colonne doit toujours être précédé du nom de la table à laquelle elle appartient.

Si le nom de la table contient des espace, encapsuler le nom de la table entre guillement : 

'nom table'
'Vente mens.[nomColonne]'

NomTable[NomColonne]

Nom de mesure = <NomFonction>(nomTable[nomColonne])
Total sales   = SUM(Ventes[Revenu total])

Groupe de mesures​

L'objectif des groupes de mesures est d'organiser vos mesures. Vous ne voulez pas mélanger vos mesures avec vos attributs dans les tables.

Vous pouvez créer autant de groupes de mesures qui nécessaire.

Créer un un groupe de mesures​

  • Créer une table de données vide via Power BI Desktop

  • Donner un nom significatif pour le groupe de mesure

    Cela ajoute la table vide dans le modèle de données avec une colonne vide : "Column1"

  • Déplacer les mesures dans la nouvelle table créée

    1. Sélectionner une mesures à déplacer

    2. Sélectionner l'onglet Measure tools

    3. Repérer l'icone Home table

    4. Cliquer sur la liste déroulante pour afficher les tables disponibles

    5. Sélectionner la table cible pour y déplacer la mesure

  • Supprimer la colonne créer automatiquement lors de la création de la table

  • Fermer & ouvrir

  • Cela donne un groupe de mesures, avec l'icone de mesure devant le libellé

Formater le code DAX​

Ne pas écrire votre formule sur une seule ligne. Mettre le nom de la mesure avec le = sur une ligne et commencer le code sur la ligne suivante

Mesure nom = 
Code DAX...

Règles générale:

  • Mettre une seule fonction DAX par ligne
  • Pour les fonctions complexes, mettre le détail des options sur les lignes suplémentaires avec une indentation.
  • Fermer la fonction avec la parenthèse de fin ")" sur une ligne aligné avec la fonction qu'elle termine
Exemple de mesure = 
CALCULATE( [Quantité total],
    FILTER ( vente,
        Vente[Nom index client] > 5 &&
        Vente[Nom index client] < 20
        ),
        FILTER( Vente, vente[Channel] = "Wholesale")
    )

Variable​

Les variables sont défini au début du code et son réutilisé par la suite.

Noter que les variables sont calculé une seule fois.

Syntaxe:

Définir vos variable en utilisant la constant "VAR" =

Puis utiliser la fonction "RETURN" pour définir ce que votre code doit retourner.

Exemple : DAX sans variable

Exemple de mesure = 
CALCULATE( [Quantité total],
    FILTER ( vente,
        Vente[Nom index client] > 5 &&
        Vente[Nom index client] < 20
        ),
        FILTER( Vente, vente[Channel] = "Wholesale")
    )

Exemple : DAX avec variable

Utiliser les variable dans le code

Exemple de mesure avec variable = 
VAR CustomerMin = 5
VAR CustomerMax = 20
VAR CustomerChannel = "Wholesales"

RETURN
CALCULATE( [Quantité total],
    FILTER ( vente,
        Vente[Nom index client] > CustomerMin &&
        Vente[Nom index client] < CustomerMax
        ),
        FILTER( Vente, vente[Channel] = CustomerChannel)
    )

Exemple : DAX avec variable

Référence des variables dans des variables

Exemple de mesure avec variable = 
VAR CustomerMin     = 5
VAR CustomerMax     = 20
VAR CustomerChannel = "Wholesales"
VAR CustomerRange   = FILTER ( vente, 
                        Vente[Nom index client] > CustomerMin &&
                        Vente[Nom index client] < CustomerMax
                    )
VAR Channel         = FILTER( Vente, vente[Channel] = CustomerChannel)

RETURN
CALCULATE( [Quantité total], CustomerRange, Channel )

La mesure devient plus simple à lire et plus facile à maintenir

Chaque bout de code est maintenant plus simple.

Mesure rapide​

La fonction mesure rapide permet de créer rapidement et simplement des mesures, mais, cela ne vous préviens pas des erreurs.

Si vous utilisez cette fonction sans vraiment comprendre comment la fonction DAX fonctionne, vous pouvez faire des erreurs sans vous en rendre compte.

La fonction mesure rapide utilise parfois des fonctions DAX que vous n'avez pas peut-être pas besoin dans dans votre mesures.

Mesure DAX générer par la fonction mesure rapide:

Total Sales average per channel =

​ AVERAGEX( ​ KEEPFILTES( VALUES( vente[Channel] ) ) , ​ CALCULATE ([Vente total] ) ​ )

Le code en rouge est inutile :

​ Mesures DAX vraiment requis :

​ Total Sales average per channel =

​ AVERAGEX( VALUES( vente[Channel] ), [Vente total] )

Cette fonction demeure intéressante, mais dans un optique d'apprentissage pour creuser plus loin comment utiliser les fonctions utiliser.

DAX et relation dans le modèle de données​

Afin que les fonctions DAX puisse être utilisé correctement, il est primordiale que les relations entre les tables dans le modèle de données soient correctement créées.

Dans le cas où une relation n'est pas défini, les calcul DAX seront erronés ou ne fonctionneront pas du tout

Sans les relations, une mesures sur la table "Sales" ne peux pas être filtrer ou segmenter par les attributs des tables

  • Regions
  • Products
  • Customers

Au même titre, la mesure suivant ne fonctionnera pas sans relation car elle doit avoir une relation entre Sales/vente et Customer/client afin de pouvoir effectuer le filtre sur le champ Vente[Nom index client]. 

Exemple de mesure avec variable = 
VAR CustomerMin     = 5
VAR CustomerMax     = 20
VAR CustomerChannel = "Wholesales"
VAR CustomerRange   = FILTER ( vente, 
                        Vente[Nom index client] > CustomerMin &&
                        Vente[Nom index client] < CustomerMax
                    )
VAR Channel         = FILTER( Vente, vente[Channel] = CustomerChannel)

RETURN
CALCULATE( [Quantité total], CustomerRange, Channel )

Pratiquer est la seule vrai manière d'apprendre​

DAX est un langage qui nécessite de comprendre et assimiler son comportement. Ce n'est pas seulement un nouveau language de programmation, c'est aussi une manière de penser.

Et le façon la plus facile de l'assimiler, est de pratiquer.

Lire un livre ou un blog sur le DAX est insuffisant. Il faut ouvrir Power BI Desktop et coder des mesures et essayer vos calcul dans des visuels sur un jeux de données simple que vous connaissez.

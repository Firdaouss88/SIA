Introduction à DAX Studio​

Dax Studio est un outil externe qui peut être connecté à Power BI Desktop, Power Pivot ou Sql Server Data Tool. Il est utilisé pour tester les requêtes et vérifier la performance des mesures crées. Par conséquent, si vous utilisez Excel et vous souhaitez vérifier les performances d'une mesure exécutée dans une requête MDX, vous pouvez récupérer la requête MDX EXCEL et la lancer dans l'outil DAX Studio afin d'en obtenir sa performance.

Connexions​

Lorsque vous lancez DAX Studio, en général, la première chose que vous obtenez est la page de connexion à un modèle à l'aide de trois options : modèle PowerPivot, Power BI ou SQL server data tool. Il existe également l'option avancée pour se connecter à votre modèle :

![ ](s:\PowerBi-EODE_nePasEffacer\Formation Power BI\Mardown\PBI Formation\Formation Dax Studio\Image\Connect2.png)

  1. Power Pivot Model : Vous remarquez que la première option est désactivée, la raison est que si vous voulez vous connecter à DAX Studio via un modèle PowerPivot d'un fichier Excel, vous devez démarrer DAX Studio à partir du fichier Excel plutôt qu'à partir du Power BI : ![](S:\PowerBi-EODE_nePasEffacer\Formation Power BI\Mardown\PBI Formation\Formation Dax Studio\Image\Connect-excel.png)

  1. PBI/SSDT MODEL : La 2ème option est de se connecter via Power BI. Vous aurez alors une liste qui contient tous les fichiers Power BI ouverts sur votre machine. Vous choisissez le fichier auquel vous voulez connecter votre modèle. Il existe une autre manière de se connecter à un modèle Power BI. Lorsque vous avez le fichier Power BI ouvert, vous cliquez sur Outils Externes puis sur DAX Studio. Vous aurez alors l'outil DAX Studio connecté automatiquement à votre modèle : ![](S:\PowerBi-EODE_nePasEffacer\Formation Power BI\Mardown\PBI Formation\Formation Dax Studio\Image\Connect1.png)

    L'image ci-dessous montre la connexion à votre modèle qui se trouve à gauche :

    ![](S:\PowerBi-EODE_nePasEffacer\Formation Power BI\Mardown\PBI Formation\Formation Dax Studio\Image\Connect1-2.png)

  1. Tabular Server : La troisième option pour connecter DAX Studio à un modèle est celle d'utiliser un serveur. Le serveur peut être un serveur Azure Analysis Services ou un serveur sur votre machine local.

  1. Advanced Options : La dernière option de connecter DAX Studio à un modèle est via les options avancées :

    1. Initial Catalog : est la base de données à laquelle DAX Studio se connecte immédiatement dans le cas où vous êtes administrateur modèle.
    2. Direct Query Mode : est dans le cas où vous aviez une requête directe et celle-ci était utile pour les anciennes versions d'analysis Services.
    3. MDX Compatibility : active le mode de compatibilité spécifique qui était utile avec la version antécédente d'Excel.
    4. Effective User Name / Rôle / Local : est dans le cas où vous êtes administrateur de la base de données. Vous pouvez ouvrir une connexion afin de vérifier la compatibilité d'une requête. Vous pourrez alors vérifier si elle a été exécutée par un autre utilisateur ou sous un rôle spécifique.

Une fois que vous ouvrez DAX Studio, vous pouvez commencer à travailler avec la base de données (Modèle) en utilisant les fonctionnalités disponibles dans le panneau Métadonnées (à gauche de la dernière image au dessus de la section l'onglet Métadonnées) .

Panneau de métadonnées​

Onglet Métadonnées​

Le panneau de métadonnées vous offre la possibilité de sélectionner la base de données sur laquelle vous souhaitez travailler, les colonnes et les différentes mesures. Le panneau vous permet également de consulter toutes les fonctions disponibles dans DAX. ![](S:\PowerBi-EODE_nePasEffacer\Formation Power BI\Mardown\PBI Formation\Formation Dax Studio\Image\Panneau-Metadata1.png)

Combo A : choisir la base de données Combo B : choisir la perspective (si la base contient "multiples perspectives", choisissez "Model") Combo C : faire la recherche pour trouver un nom spécifique dans votre modèle.

Si vous survolez votre flèche une colonne ou une table, vous aurez le détail du contenant.

​ Par exemple : ![](S:\PowerBi-EODE_nePasEffacer\Formation Power BI\Mardown\PBI Formation\Formation Dax Studio\Image\DAX2.png)

Onglet Fonctions​

L'onglet Fonctions active la liste de toutes les fonctions DAX qui sont pris en charge par la base de données à laquelle vous êtes connecté.

![](s:\PowerBi-EODE_nePasEffacer\Formation Power BI\Mardown\PBI Formation\Formation Dax Studio\Image\Fonctions.png)

Cet onglet fournit également des informations sur la syntaxe.

​ Prenons par exemple ADDCOLUMNS : ![](S:\PowerBi-EODE_nePasEffacer\Formation Power BI\Mardown\PBI Formation\Formation Dax Studio\Image\Functions-detail.png)

Onglet DMV​

L'onglet DMX est plutôt technique, son utilisation n'est donc pas très fréquente. Il s'agit d'un type spécial de requêtes exécutées sur un serveur pour recueillir des informations spécifiques.

![](S:PowerBi-EODE_nePasEffacer\Formation Power BI\Mardown\PBI Formation\Formation Dax Studio\Image\DMV.png)

Ruban et son contenu​

Toutes les fonctionnalités de DAX Studio sont disponibles via le ruban. Vous pouvez utiliser n'importe qu'elle fonctionnalité en cliquant sur l'icone de votre besoin :![](S:\PowerBi-EODE_nePasEffacer\Formation Power BI\Mardown\PBI Formation\Formation Dax Studio\Image\Ruban.png)

  1. RUN : Exécuter une requête.
  2. Cancel : Annuler une requête qui a pris beaucoup de temps à s'exécuter.
  3. Clear Cache : Vider le cache du modèle avant d'exécuter la requête.
  4. Output : Choisir l'affichage de votre résultat de requête selon différentes manières, par exemple vers Excel, CSV, Bloc notes ou autres.
  5. Query Builder : Générateur de requêtes qui vous permet de créer une requête via une interface graphique si vous n'êtes pas familier avec DAX ou si vous souhaitez créer une requête rapidement.
  6. Undo/Redo : Options d'édition, annuler, refaire, copier et couper.
  7. Format Query : Option de formatage afin que vous puissiez formater automatiquement votre code.
  8. To Upper/To Lower : Changer une chaîne en majuscule ou en minuscule.
  9. Comment/Uncomment : Commenter ou décommenter des morceaux de code.
  10. Find/Replace : Options de recherche et de remplacement.
  11. Load Perf Data : Charger les données de performances qui ont été créées avec Power BI Desktop, si vous voulez analyser la performance d'un rapport ou d'une page spécifique.
  12. ALL Queries/Query Plan/Server Timings : Si vous êtes intéressé à faire l'analyse d'une requête spécifique ou de nombreuses requêtes, vous pouvez soit tracer toutes les requêtes, activer le plan de requête pour voir les informations détaillées sur l'exécution de votre requête ou regarder les horaires du serveur pour avoir des mesures précises du temps qui a été dépensé dans différentes parties du moteur lors de l'exécution de la requête.
  13. Connect/Refresh Metadata : Permet de réouvrir la boîte de dialogue de connexion et vous pouvez aussi actualiser les métadonnées en ramenant la dernière version disponible sur le serveur.

Il existe un onglet Avancée sur le ruban qui est : ![](S:\PowerBi-EODE_nePasEffacer\Formation Power BI\Mardown\PBI Formation\Formation Dax Studio\Image\Advanced.png)

  1. Import/Export/View Metrics : Importer exporter les métriques qui sont des informations importantes sur la taille de votre modèle, la taille de chaque colonne pour analyser et déboguer un modèle.

  2. Export Data : Exporter des données sur des fichiers CSV ou sur un serveur SQL.

  3. RUN Benchmark : Exécuter un test de performance si vous voulez mesurer la performance de la requête.

  4. SQL Profiler/Analyze in Excel : Ouvrir des portes externes comme SQL Profiler ou analyser dans Excel avec le même modèle où vous êtes connectées sur DAX Studio en cliquant sur leur icone respective.

    Exécuter une requête ( F5 )​

Vous pouvez rédiger une requête et l'exécuter avec le triangle vert RUN ou F5 et vous aurez le résultat dans le panneau résultats. Il est possible d'augmenter son affichage ou de le diminuer en utilisant le bouton Ctrl+ et Ctrl- ou en maintenant Ctrl enfoncé et en défilant la molette de votre souris :

![](S:\PowerBi-EODE_nePasEffacer\Formation Power BI\Mardown\PBI Formation\Formation Dax Studio\Image\Query execute.png)

Il existe aussi un onglet en bas de la page qui présente l'historique des requêtes que vous avez exécuté. Vous avez l'heure de début, le nombre de lignes renvoyées, les millisecondes exécutées par heure et par le serveur ainsi que le nom du serveur. Si vous souhaitez exécuter une requête antécédente, il suffit de cliquer deux fois sur le contenu de la colonne Query et la requête sera affichée en haut. Vous pouvez finalement l'exécuter avec RUN ou F5 :

![](S:\PowerBi-EODE_nePasEffacer\Formation Power BI\Mardown\PBI Formation\Formation Dax Studio\Image\Query History.png)

Chaque fois que vous exécutez une requête une partie de la requête est mise en cache et conservée dans des mémoires déjà calculées. Cela permet au moteur de récupérer la même requête déjà exécutée plutôt que de recalculer les informations, ce qui augmente la vitesse d'affichage. C'est utile, mais si vous voulez faire une analyse des performances, vous n'aurez pas besoin de cache pour avoir des données cohérentes et résultats correctes, c'est pourquoi nous vous suggérons de vider le cache avant d'effectuer une analyse de performances.

Il est possible de vider le cache et d'exécuter la requête en un seul clic en sélectionnant "Clear Cache then Run" sous le bouton "Run" :

![](S:\PowerBi-EODE_nePasEffacer\Formation Power BI\Mardown\PBI Formation\Formation Dax Studio\Image\Clear cache.png)

Résultat d'une requête​

Si vous désirez avoir le résultat de votre requête dans un fichier Excel, une grille ou autres, choisissez le type d'Output que vous désirez puis cliquez sur Run :

![](S:\PowerBi-EODE_nePasEffacer\Formation Power BI\Mardown\PBI Formation\Formation Dax Studio\Image\OUTPUT.png)

Générateur de requête​

Un générateur de requêtes (Query Builder) vous fournit une interface utilisateur graphique qui vous facilite la création des requêtes. Dans la partie Builder, vous pouvez glisser vos colonnes et vos champs calculés ce qui vous permet ainsi de créer des mesures et de les rajouter dans l'affichage comme l'exemple suivant :

![](S:\PowerBi-EODE_nePasEffacer\Formation Power BI\Mardown\PBI Formation\Formation Dax Studio\Image\Builder.png)

La mesure Margin est crée en cliquant sur le bouton NEW. Écrivez ensuite le code DAX que vous désirez : ![](S:\PowerBi-EODE_nePasEffacer\Formation Power BI\Mardown\PBI Formation\Formation Dax Studio\Image\MEsure.png)

Si vous êtes intéressé par le code généré derrière ces créations ainsi que vos champs glissés dans votre Builder, et que vous désirez sauvegarder cette requête quelque part, vous devez cliquer sur Edit Query et la requête apparaîtra : ![](S:\PowerBi-EODE_nePasEffacer\Formation Power BI\Mardown\PBI Formation\Formation Dax Studio\Image\Edit Query.png)

Formater code DAX​

Dans DAX, un point crucial est de formater votre code, c'est pourquoi l'outil DAX Studio a rajouté cette option dans son ruban, afin de faciliter la tâche aux utilisateurs et rendre son utilisation simple . Cette option est directement liée au site daxformatter.com qui est un outil gratuit de SQLBI et qui transforme vos formules DAX brutes en code propre, beau et lisible. ![](S:\PowerBi-EODE_nePasEffacer\Formation Power BI\Mardown\PBI Formation\Formation Dax Studio\Image\FomatterDAX.png)

Vous avez deux options de formatage. Vous pouvez utiliser la première option qui est le format de ligne courte ou la deuxième, soit le format de ligne longue, dépendamment de la longueur de votre code.

Outils d'édition​

DAX Studio contient une section dans son ruban d'outil d'édition dédié à faciliter la modification de votre code. Vous avez ainsi la possibilité de commenter un bout de code ou le décommenter, fonction utile dans le débogage de votre code. Vous pouvez choisir le délimiteur virgule ou point-virgule comme séparateur de liste. Vous pouvez également fusionner des paramètres XML et utiliser le chercher et remplacer.

![](S:\PowerBi-EODE_nePasEffacer\Formation Power BI\Mardown\PBI Formation\Formation Dax Studio\Image\Edition.png)

Charger les données de l'analyseur de performances​

DAX Studio vous propose également des fonctionnalités pour analyser la performance des requêtes DAX. Vous devez tout d'abord générer avec Power BI les données que vous voulez analyser pour ensuite suivre les étapes dans l'image ci-dessous :

![](s:\PowerBi-EODE_nePasEffacer\Formation Power BI\Mardown\PBI Formation\Formation Dax Studio\Image\Performance.png)

Une fois sur votre page vous choisissez l'élément à analyser, vous cliquez sur Exporter et vous enregistrez vos données sous format JSON File: ![](S:\PowerBi-EODE_nePasEffacer\Formation Power BI\Mardown\PBI Formation\Formation Dax Studio\Image\JSON File.png)

Dès que le fichier est enregistré, vous pouvez l'ouvrir par DAX Studio à partir de l'option Load Perf Data puis sélectionnez votre fichier JSON et vous obtenez :

![](S:\PowerBi-EODE_nePasEffacer\Formation Power BI\Mardown\PBI Formation\Formation Dax Studio\Image\Load Perf.png)

Une fois le fichier JSON ouvert,le panneau inférieur présentera la requête à démarrer. Lorsque cette dernière a terminé, le nombre de lignes renvoyées, le nombre de millisecondes pour exécuter la requête à rendre et le temps total d'exécution vous seront présentés.

Si vous voulez inspectez la requête, il vous suffit de faire un double-clique sur la colonne requête et ensuite formater votre code afin qu'il soit lisible (voir section Formater code DAX pour le détail sur comment formater).

Capture des requêtes exécutées sur un serveur​

Dans DAX Studio vous avez l'option ALL QUERIES. Cette dernière, si elle est activée (juste en cliquant sur l'icone), vous permet d'activer le suivi de toutes les requêtes en cours d'exécution sur Power BI. Vous obtiendrez ainsi pour chaque requête son heure de début exécution, le type de la requête, la durée, l'utilisateur qui l'a exécuté, la base de données ainsi que la requête. ![](S:\PowerBi-EODE_nePasEffacer\Formation Power BI\Mardown\PBI Formation\Formation Dax Studio\Image\Durée Query.png)

Cette option de tracer toute les requêtes est très utile pour optimiser ou analyser vos données de performances.

Plan de requête et horaires du serveur​

Lorsque vous voulez optimiser une seule requête, vous devez activer le plan de requête et les horaires du serveur afin de recueillir des informations précises sur l'endroit où la requête a passé plus du temps .

![](S:\PowerBi-EODE_nePasEffacer\Formation Power BI\Mardown\PBI Formation\Formation Dax Studio\Image\Time Query.png)

Une fois que vous activez vos deux options (1) et que vous exécutez votre requête (2), vous obtenez deux onglets qui s'ouvrent en bas de la page et qui sont Query Plan et Server Timings :

![](S:\PowerBi-EODE_nePasEffacer\Formation Power BI\Mardown\PBI Formation\Formation Dax Studio\Image\Onglets.png)

  • Server Timing : les informations fournies sont le montant total en millisecondes de l'exécution de la requête entre le processus de formule (FE) et le processus de stockage (SE), l'information de combien de millisecondes a utilisé le CPU ainsi que la partie où il y a le détail de la requête, la durée, le temps CPU ,le nombre de lignes, le kilo-octet du cache de données qui a été renvoyé (KB) et aussi la colonne requête : ![](S:\PowerBi-EODE_nePasEffacer\Formation Power BI\Mardown\PBI Formation\Formation Dax Studio\Image\ST.png)
  • Query Plan : Le plan de requête est divisé en deux parties soit la partie physique et la logique. Le plan physique étant extrêmement long et difficile à lire, ne parcourez que les lignes nécessaires. Le plan logique est légèrement plus simple, il est préférable de le parcourir lorsque vous avez une requête complexe. ![](S:\PowerBi-EODE_nePasEffacer\Formation Power BI\Mardown\PBI Formation\Formation Dax Studio\Image\Plan query.png)

Exécuter le benchmark​

Le Benchmark vous permet d'exécuter la même requête plusieurs fois, à la fois sur cache froid et cache chaud (définition au dessous de l'image), afin d'avoir une meilleure interprétation des données . ![](S:\PowerBi-EODE_nePasEffacer\Formation Power BI\Mardown\PBI Formation\Formation Dax Studio\Image\BEnchmark.png)

Cache froid : Lorsque le cache est vide ou contient des données non pertinentes, de sorte que CPU doit effectuer une lecture plus lente à partir de main memory pour vos besoins en données de programme.

Cache chaud : Lorsque le cache contient des données pertinentes et que toutes les lectures de votre programme sont satisfaites à partir du cache lui-même.

Une fois que vous cliquez sur Run (2), vous obtenez un résumé de l'exécution : ![](S:\PowerBi-EODE_nePasEffacer\Formation Power BI\Mardown\PBI Formation\Formation Dax Studio\Image\Benchmark1.png)

Le résumé contient des valeurs en millisecondes, la durée totale, l'écart type, le minimum, le maximum et la moyenne pour une exécution dans le cache froid et une autre dans le cache chaud.

Afficher, importer, exporter des métriques​

Dans l'onglet Advanced de DAX Studio, vous avez les métriques. Pour afficher les métriques d'un modèle il suffit de cliquez sur View Metrics et vous aurez les informations sur chaque table, colonne, relation, nombre de lignes, taille des données, taille de la table, dictionnaire, type d'encodage et autres. Ces informations sont très importantes lors de l'optimisation d'un modèle.

![](S:\PowerBi-EODE_nePasEffacer\Formation Power BI\Mardown\PBI Formation\Formation Dax Studio\Image\View metrics.png)

![](S:\PowerBi-EODE_nePasEffacer\Formation Power BI\Mardown\PBI Formation\Formation Dax Studio\Image\Relation modele.png)

Ces informations, que vous obtenez à partir de View Metrics, sont enregistrables afin que vous puissiez les utiliser plus tard dans DAX Studio ou l'outil VertiPaq Analyser. Pour ce faire, vous allez cliquer sur Export Metrics et enregistrer ces résultats sous un fichier JSON. ![](S:\PowerBi-EODE_nePasEffacer\Formation Power BI\Mardown\PBI Formation\Formation Dax Studio\Image\Export metrics.png)

Si vous avez besoin d'aide sur votre modèle, vous pouvez envoyer la version Metrics que vous venez d'exporter afin que le destinataire puisse regarder votre modèle et vous propose des suggestions ou corrections.

Vous avez la possibilité d'importer une métrique sans vous connecter à une base de données ou un modèle : ![](S:\PowerBi-EODE_nePasEffacer\Formation Power BI\Mardown\PBI Formation\Formation Dax Studio\Image\Import Metrics.png)

Options DAX Studio​

Pour modifier les options de votre outil DAX Studio ,vous pouvez uniquement cliquer sur Fichier puis sur Options et vous aurez ainsi le droit de configurer le format d'exportation de données, choisir de vider le cache avant chaque exécution et autres détails qui personnalisent votre DAX Studio.

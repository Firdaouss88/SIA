Intro PBI - Requête et transformation des données

Matériels pour les exercises​

Montrer le path vers les données pour les exercises

inclus:

  • Gabarit Power PI
  • Données source Excel
  • dimDate

Contenu:

Cette formation comprend des explications sur les concepts de base de Power BI et de préparation des modèles de données.

1 - Éditeur de requête

La première chose à faire pour créer des rapports est de charger des données.

Bien que l'option la plus évidente soit de sélectionner "1- Obtenir des données", ce n'est pas recommandé.

Il est mieux d'utiliser l'option 2 -Transformer les données.

L'éditeur de requête est un outil à l'intérieur de Power BI qui :

  • Permet de transformer vos données de la forme brute à une forme optimisée pour Power BI ;
  • Est l'endroit où vous passerez 90 % de votre temps.

Comment charger des données​

  1. Sélectionner Nouvelle Source
  2. Sélectionner Excel

  1. Naviguer jusqu'au fichier Excel à charger

  2. Sélectionner le fichier à charger

  1. L'éditeur présente le navigateur avec le contenu du fichier Excel disponible pour charger dans Power BI.
  2. Différents icones sont présentés :
    • Table nommée : Une table spécifique dans Excel avec un nom
    • Onglet ou page Excel: Une page Excel

On recommande de toujours utiliser des tables nommées.

La raison est simple, cela simplifie le travail de la requête pour retrouver les données dans Excel et limite les erreurs de chargement.

  1. Sélectionner les données à charger

  1. Cela transfert les données du fichier Excel dans le fichier Power BI

Les données sont maintenant disponibles pour être transformées et améliorées

  1. L'onglet "Transformer" donne accès à plusieurs options et un clique droit sur n'importe quelle colonne offre encore plus de possibilité de transformation

Comment transformer des données Excel en tableau nommé​

  1. Sélectionnez une cellule parmis les données que vous voulez transformer en tableau

  2. Sélectionnez l'onglet "insertion"

  3. Cliquer sur l'option tableau

  4. Cliquer sur OK pour transformer la plage de données en tableau nommé

  1. Les données sont transformées en table et le visuel change pour le style de tableau actif

  2. L'onglet création est maintenant disponible et offre plusieurs options pour personnaliser le tableau

  3. Il est maintenant possible de nommer le tableau.

    C'est ce nom qui sera affiché lors du chargement dans Power BI.

    Un avantage important des tableaux est que lors du chargement, Power BI ignorera les valeurs dans les colonnes en dehors de la zone du tableau.

2 - Transformation de base

Sélectionner seulement les attributs utilisés​

L'objectif est de charger dans Power BI seulement les colonnes et tables qui sont vraiment utilisées dans le rapport.

Power BI étant un outils qui garde les données en mémoire, le fait de retirer tout attribut non utilisé a un impact important :

  • Réduit la mémoire nécessaire
  • Augmente la performance
  • Simplifie le modèle
  • Améliore et simplifie l'utilisation du jeu de données et la création de rapport

Il est important de comprendre qu'il est presqu'impossible de connaitre exactement les colonnes qui seront utilisées au début d'un projet. Il est donc fort probable que cette étape soit exécutée plusieurs fois à des moments différents.

Plus le nettoyage des sources est exécuté tôt, plus les impacts sur le reste des actions pour nettoyer les données sont réduit significativement.

Voici les étapes à suivre:

  1. Dans la section Requêtes, sélectionnez une table

  2. Dans le section Étapes appliquées, sélectionnez Navigation

    ll est important de sélectionner les colonnes le plus tôt possible dans les étapes appliquées afin de ne pas charger des données inutilement.

  3. Dans le Ruban, sélectionnez l'option Choisir les colonnes

  4. Cliquez sur insérer

  5. Décochez les colonnes non utilisées dans le rapport

Il existe d'autres options pour retirer les colonnes non utilisées, tel que supprimer, mais l'option Sélectionner les colonnes est plus efficace car elle filtre les colonnes à la source et ne les charge pas du tout dans le Power BI.

Nettoyer les noms des données​

La première étape après avoir chargé les données brutes est de nettoyer le nom de vos données.
L'objectif est d'avoir des noms logiques pouvant être interprétés correctement par tous.

  • Pas d'abréviation
  • Pas de souligné*
  • Pas de CamelCase (première lettre des mots en majuscule et tous les mots collés)

La seule exception est les noms des attributs techniques utilisés dans le modèle pour créer les relations. Il s'agit de noms qui sont normalement masqués et non visibles aux utilisateurs.

Modifier le nom des tables​

  1. Dans la section requêtes, double-cliquez sur le nom d'une table
  2. Dans la section requêtes, faites un clique droit et sélectionnez l'option Renommer
  3. Dans la section Propriété, sous Nom, modifiez le nom existant ou saisissez le nom désiré

Modifier le nom des attributs​

  1. Sélectionnez une table
  2. Révisez une à une chacune des colonnes pour ajuster le libellé

Noter les attributs ou nom de colonne inutile​

Il est important de prendre en note le nom des colonnes ou attributs que vous avez identifiés comme étant inutiles pendant la révision des libellés.

Fusionner des sources en une seule table​

Lorsque plusieurs sources ont la même structure de données, par exemple 3 onglets d'un fichier Excel:

  • Ventes 2018
  • Ventes 2019
  • Ventes 2020

On ne veut pas avoir 3 tables différentes dans notre modèle, mais plutôt une seule table des ventes qui contient toutes les données de toutes les années

Étapes à suivre:

  1. Dans la section requêtes, sélectionnez la première table de données de vente.

  2. Dans le Ruban, onglet Accueil, sélectionnez l'option Ajouter des requêtes.

  3. Dans le fenêtre Ajouter, sélectionnez l'option Au moins trois tables et sélectionnez les tables à fusionner avec la première.

  4. Dans la section requêtes, double-cliquez sur la table source de la fusion et ajustez le nom.

  5. Cette table contient maintenant les données des années 2019 et 2020.

  6. Notez l'ajout d'une étape dans le section Étapes appliquées : Requête ajoutée.

  7. Notez icone de tri dans le nom de la colonne "Purschase Date" pour indiquer un tri de décroissant et l'étape supplémentaire Ligne triées dans la section étapes appliquées.

    Le tri est appliqué seulement pour démontrer que les ventes de l'années 2020 sont maintenant présentes dans la table des ventes 2018

Désactiver le chargement des sources non utilisés dans le rapport​

Une fois que les ventes de 2019 et 2020 sont fusionnées avec les données de 2018, elles ne sont plus requises pour la suite des transformations ni requises dans le modèle de données cible Power BI.

Les données en tant que tel seront présentes dans la requête nommé "Sales". Les sources originales de 2019 et 2020 sont requises pour faire le fusion, mais ne sont plus nécessaires physiquement dans le modèle de données.

On peut alors désactiver le chargement de ces sources.

Désactiver le chargement des données sources:​

  1. Dans la section requêtes, sélectionnez un ou des requêtes pour lesquelles désactiver le chargement en appuyant sur Ctrl + Clique

  2. Faites un clique-droit sur une des requêtes pour afficher le menu déroulant et désélectionnez l'option Activer le chargement

Le nom de la requête devient alors en italique afin d'indiquer que les données de cette requête ne seront pas chargées dans le modèle de données Power BI.

Ces données demeurent disponibles pour les transformations dans l'éditeur de requêtes, mais elles ne seront pas disponibles dans le rapport pour créer des visualisations, des mesures en DAX, ni visible dans le modèle de données Power BI.

On peut maintenant voir les données comment étant des requêtes de travail pour les transformations dans l'éditeur de requête.

Créer un groupe de requêtes​

L'éditeur de requêtes peut rapidement devenir chargé. Entre les requêtes sur les données sources, les transformations, les fusions et autres requêtes requises pour adapter et transformer les sources de données brutes au format optimal pour Power BI, la liste des différentes requêtes devient longue.

Il est donc recommandé d'organiser les requêtes en groupe afin d'organiser et de bien structurer l'ensemble des requêtes.

Groupes de requêtes recommandés

Voici quelques noms de groupes qui devrait couvrir l'ensemble de vos opérations de transformation.

  • Paramètre

    Contient les requêtes générées par paramètre, comme la dimension date

  • Support ou source de travail

    Contient l'ensemble des requêtes sur des sources de données qui ne sont pas chargées dans le modèle Power BI

  • Travail ou transformation

    Contient l'ensemble des requêtes requises pour transformer et adapter les sources de données

  • Fait

    Contient la ou les tables finales du modèles qui seront la base des mesures et sujet d'analyse. (table de fait)

  • Dim

    Contient l'ensemble des axes d'analyses utilisés pour filtrer, grouper ou segmenter les sujets d'analyses (dimension)

Créer un groupe de requêtes​

  1. Dans la section Requêtes, sélectionnez une requête pour laquelle désactiver le chargement et faites un clique droit

  2. Dans le menu déroulant, sélectionnez l'option Déplacer vers le groupe / Nouveau groupe...

  3. Dans la fenêtre Nouveau groupe, saisissez le nom du groupe à créer et cliquez sur OK pour créer le groupe

  1. Un nouveau groupe est créé et apparaît avec l'icone de répertoire
  2. Les requêtes associées au groupe sont visibles sous le nom du groupe

Ajouter une requête dans un groupe Existant​

Pour ajouter ou déplacer une requête dans un groupe existant, il suffit de la déplacer dans le groupe cible à l'aide de la souris.

  1. Dans la section Requêtes, sélectionner une ou plusieurs requêtes avec Ctrl + clique.
  2. Glisser la/les requêtes dans le groupe cible.

Renommer un groupe de requête​

Pour renommer un groupe de requêtes, il suffit de faire ou double clique sur le nom ou de sélectionner l'option Renommer dans le menu du clique droit

Ajuster le type de données​

Power BI est très efficace pour déterminer le format des données, malgré tout il est important de valider que chacune des colonnes est au bon format.

Vérifiez ces erreurs fréquentes :

  • Champ numérique au format texte ;
  • Date avec les heures ;
  • Date au format texte ;
  • Format avec décimal lorsque non requis.

Changer le format d'une colonne​

  1. Cliquez sur l'icône de format à gauche de nom de la colonne
  2. Sélectionnez le format désiré dans le menu déroulant

6 - Créer un code à partir du texte d'une colonne​

Imaginez que vous voulez utiliser les premiers caractères d'une colonne pour créer une nouvelle colonne qui serait un code ou une abréviation.

  1. Faites un clique droit sur la colonne pour laquelle créer une abréviation et sélectionner l'option Duplication de la colonne

  1. Faites un clique droit sur la colonne copiée et sélectionnez l'option Par nombre de caractères

    À noter que les autres options aussi servent à séparer un champ et offrent des options différentes pour y parvenir.

  1. Saisissez le nombre de caractères désiré pour représenter le code ou l'abréviation

  2. Sélectionnez l'option de fractionnement

    • Une fois le plus à gauche : Sélectionne le nombre de caractères saisi en partant de la gauche pour créer une colonne et le reste des caractères dans une autre colonne
    • Une fois le plus à droite possible : Sélectionne le nombre de caractères saisi en partant de la droite pour créer une colonne et le reste des caractères dans une autre colonne
    • À plusieurs reprises : Crée autant de colonnes que nécessaire par groupe de caractères saisi.

  1. Cela crée 2 colonnes

    La première correspondant au nombre de caractères sélectionné

    La seconde avec le reste des caractères

  1. Double cliquez sur le nom de la colonne pour la renommer

  1. Faites un clique droit sur la seconde colonne créée et sélectionnez l'option Suppimer

  1. Dans la section Étapes appliquées, l'on voit l'ensemble des opérations générées pour créer la nouvelle colonne d'abréviation

    À chaque fois que les données seront rafraichies, ces étapes seront réexécutées automatiquement

Créer une table de date​

Un table de date est très importante dans un modèle de données. Cela permet d'utiliser les fonctions de time intelligence et de présenter les données correctement dans le temps dans les rapports.

Le Gabarit .pbit (Power BI desktop) fourni par l'équipe EODE contient déjà une table de dates, nommée Date.

  1. Dans l'éditeur de requête, sous le groupe de requêtes Paramètre, sélectionner Générateur de dimDate
  2. Saisir la date de début et de fin utilisée pour générer le calendrier de dates.

Transformation avancée

L'éditeur de requêtes offre plusieurs options pour transformer et adapter vos données.
Voici les moyens d'accéder au différentes options de transformation :

1 - Faire un clique droit sur une colonne ouvre un menu déroulant permettant d'interagir sur les données de la colonne.

  1. L'onglet Transformer présente les même options que le clique droit sur une colonne.

  1. L'onglet Ajouter une colonne présente les options pour ajouter de nouvelles colonnes à vos données.

  1. Un clique gauche sur l'icone situé en haut à gauche présente les options qui permettent d'affecter l'ensemble de la table.

Utiliser une source de données WEB​

Power BI permet d'utiliser des sources de données web de la même façon qu'une source provenant d'un fichier Excel ou d'une base de données.

Pour intégrer des sources WEB, il suffit de connaitre l'URL.

  1. Dans un browser web, par exemple Google, chercher California population

  1. Chercher le lien suivant :

Cliquer sur le lien pour ouvrir la page et copier l'adresse URL.

  1. Dans Power BI, ouvrir l'éditeur de requête

  1. Dans l'éditeur de requête, dans l'onglet Accueil, sélectionner l'option Nouvelle Source

  2. Parmi les sources de données affichées, sélectionner Web.

  1. Dans la fenêtre qui s'ouvre, copier l'URL trouvé précédemment.

  1. Dans la fenêtre du navigateur, sélectionner l'option Table pour afficher les données
  2. Valider le format des données et cliquer sur OK pour les charger dans Power BI.

  1. Dans l'éditeur de requêtes, dans la section Requêtes, sélectionner la nouvelle requête

  2. Dans la section Propriétés, modifier le nom pour Californie Population par ville

  1. Dans la section Requêtes, déplacer la nouvelle requête dans un groupe de mesures Support avec les tables Sales_2019 et Sales_2020

Créer un Lookup ou une jointure sur une autre table​

L'une des fonctions les plus utilisée dans Excel est le VLOOKUP. Cette fonction permet d'utiliser un champ dans un onglet d'Excel, de retrouver ce champ dans un autre onglet et de retourner chacune des données d'une colonne différente pour l'ajouter au premier onglet.

Avec Power Query, la fonction équivalente se nomme Merge Query. Il suffi d'avoir deux champs identiques entre deux tables pour pouvoir les lier. Une fois liés, il est possible d'ajouter une ou des colonnes de la seconde table dans la première.

Par exemple, dans la table location, nous avons la colonne Name qui correspond à la colonne City de la source Web Californie population par ville. Nous allons utiliser ces deux champs pour lier la table Location avec la table California population par ville et ajouter la Population de la seconde table à la première.

Noter que la fusion de requêtes ne consiste pas à créer une relation dans le modèle.

Elle sert seulement à faire une jointure temporaire afin de récupérer des données dans une table pour les ajouter à une autre.

  1. Dans la section Requêtes, sélectionner la requête dans laquelle vous voulez ajouter des colonnes : Location
  2. Dans l'onglet Accueil, sélectionner l'option Fusionner des requêtes

  1. Dans la fenêtre Fusionner, sélectionner la colonne utilisée pour faire la relation avec la première table : Name

    Cette colonne peut avoir un nom différent dans les deux tables, mais son contenu doit correspondre et avoir le même format. Si le format est différent, la correspondance sera de 0 %. Noter que la correspondance ne doit pas être de 100 % nécessairement. Dans un cas inférieur à 100 %, aucune donnée ne sera retournée pour les lignes sans correspondance dans la seconde table.

  2. Sélectionner le nom de la table avec laquelle faire la relation (jointure) :

    Cliquer sur la flèche pour faire apparaitre le menu déroulant et sélectionner la table California population par ville.

  1. Une fois la table choisie, sélectionner la colonne utilisée pour faire la relation de la seconde table : City

  1. Parce que la source de données est Web, une fenêtre apparaitra. Sélectionner Ignorer les contrôles des niveaux de confidentialité pour ce fichier... puis cliquer Enregistrer

  1. Le type de jointure est très important. C'est lui qui détermine comment la relation est évaluée et quelles données sont retournées suite à la jointure entre les deux tables. Sélectionner Externe gauche : Cela retourne 100 % des données de la première table et 100 % des données de la seconde pour les lignes avec une correspondance.

  1. Un fois la relation exécutée, un message s'affiche dans le bas de la fenêtre indiquant combien de lignes ont une correspondance dans la seconde table. Appuyer sur OK

  1. La fusion de requêtes ajoute une étape dans la section Étapes appliquées

    Une nouvelle colonne est ajoutée dans la table source et son contenu affiche Table.

  1. Cela signifie que l'ensemble des données de la seconde table peuvent maintenant être sélectionnées pour être ajoutées à la première.

  1. Cliquer sur l'icone Étendre ou Expand ouvre un menu affichant la liste des champs de la table fusionnée
  2. La majorité du temps, obtenir l'ensemble des champs n'est pas nécessaire. Il est possible de sélectionner ou désélectionner les champs selon le besoin. Pour notre exemple, conserver seulement le champ : Population

  1. Ce nouveau champ est au format texte et contient des , entre les centaines et les milliers.

    Faire un clique droit sur la nouvelle colonne Californie population par ville.Population pour afficher le menu déroulant des options de colonne.

    Sélectionner l'option Remplacer les valeurs...

  1. Dans la fenêtre Remplacer les valeurs, saisir , dans la case Valeur à rechercher et laisser la case Remplacer par vide. Cliquer sur OK

  1. Cliquer sur l'icone de format des données à gauche de la colonne Californie population par ville.Population

  2. Sélectionner Nombre entier

    Si on ne fait pas le remplacement des valeurs, la conversion ne se faire pas correctement et on un résultat erroné.

Pour faire la modélisation de données, il faut être dans Power BI. Si l'éditeur de requêtes est ouvert, fermez-le :

  1. Dans l'onglet accueil de l'éditeur de requête, cliquer sur l'icone Fermer & appliquer

Se familiariser avec les données​

L'une des premières choses que l'on veut faire est de se familiariser avec les données.

  1. La vue données représentée par une grille à gauche de l'interface permet de visualiser les données sous un format grille, comme dans Excel.

Valider les relations​

Une fois que vous avez révisé les données et que vous vous êtes familiarisé, la plus importante à faire est de valider et créer les relations entre les tables.

Il est important avant tout de valider les relations entre les tables. Si vous venez du monde Excel, ce n'est pas une tâche ou activité avec laquelle vous êtes familier, mais bien déterminer les relations entre les tables est la chose la plus importante en modélisation.

Power BI essaie de détecter les relations automatiquement, mais il fait souvent des erreurs.

Il est donc recommandé de supprimer les relations générées par Power BI et de les créer soi-même. Cela permet de:

  • S'assurer que les relations sont bonnes

  • S'assurer que le modèle reflète la vision / le besoin

  1. Dans la vue Modèle, sélectionner un lien.
  2. Power BI surligne en jaune le lien et affiche les champs utilisés pour faire la relation dans les tables correspondantes
  3. Faire un clique droit sur un lien

Supprimer une relation existante​

  1. Faire un clique droit sur un lien
  2. Sélectionner Supprimer dans le menu déroulant

Modélisation définition et trucs de bases​

Un modèle est composé d'au moins 1 table de fait et de plusieurs axes d'analyses.

Table de fait :

  • C'est la table principale qui représente le sujet d'analyse ;
  • Elle contient les champs numériques que l'on va utiliser pour faire des calculs ;
  • Elle contient un champ permettant de faire le lien avec chaque table d'axe d'analyse, dimension. Ce champ est nommé ID de l'axe d'analyse ;
    Par exemple: id_client
  • Elle ne contient aucun champ descriptif ou libellé ;
  • Power BI étant très efficace pour faire des agrégations, on favorisera des tables avec peu de colonnes et beaucoup de lignes.

Table d'axe d'analyse ou dimension

  • Ces tables contiennent les attributs utilisés pour grouper, filtrer ou segmenter la table de fait (Slice & dice) ;
  • La plupart des champs textes se retrouvent dans ces tables ;
  • Chaque description est conservée une seule fois avec un identifiant ;
  • L'identifiant se retrouve dans la table de fait et permet de faire le lien afin d'utiliser les différents libellés.

Trucs et astuce

  • On ne veut pas avoir une seule table comme dans Excel avec une centaine de colonnes.

  • On veut un table contenant les champs principaux, ceux sur lesquels on veut faire des calculs (table de fait) et des tables secondaires qui contiennent les informations de détails qui seront des tables d'axes d'analyses, ou dimension.

  • Simplifier le modèle au maximum. Les relations entre les axes d'analyse et la table de fait doivent être directes et ne doivent pas passer par une autre table intermédiaire.

    Il est parfois impossible de faire une relation directe, cela sera couvert dans une formation avancée

  • Valider la granularité des tables.

    Assurez-vous que vos relations ne dupliquent pas les données de la table principale

Format et mise en place du modèle​

  1. Placer les de dimensions, axes d'analyse dans le haut de la page
  2. Placer la table de fait principale au centre, sous les axes d'analyse.

Créer des relations - Drag & Drop​

La manière la plus simple de créer une relation est de sélectionner :

  1. Sélectionner un champ avec un Clique gauche, en conservant le clique appuyé, déplacer la souris sur la table cible, se positionner sur le champ équivalent dans la table de destination et relâcher la clique de la souris.

Créer une relation avec l'icone "Gérer les relations"​

  1. Dans le menu Accueil, cliquer sur l'icone Gérer les relations

  1. Dans la fenêtre de Gérer les relations, cliquer sur Nouveau

  1. Dans la fenêtre Créer une relation, sélectionner la table source et la table de destination pour lesquelles vous voulez créer une relation
  2. Sélectionner pour chacune des tables le champ correspondant à l'identifiant de l'axe d'analyse, ou le champ qui permet de fait la relation (VLOOKUP dans Excel). et appuyer sur OK

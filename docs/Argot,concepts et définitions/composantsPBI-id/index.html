Composants de Power BI

Qu'est-ce que Power BI​

Power BI est un ensemble de services logiciels, d'applications et de connecteurs qui fonctionnent ensemble pour transformer vos sources de données indépendantes en informations cohérentes, visuelles et sécurisées. Vos données peuvent être une feuille de calcul Excel ou un ensemble de bases de données hybrides basés sur le cloud et sur site (on premise). Power BI permet de vous connecter facilement à vos sources de données, de visualiser, de découvrir ce qui est important, et de le partager avec qui vous voulez.

Composants Power BI sur site (On Premise)​

1 - Sources de données ( On-premise & Web)​

Bases de données, fichiers et autres sources de données exploités via Power BI

Une source de données est l'endroit d'où proviennent les données d'un ensemble de données. Par exemple, les données pourraient provenir de :

  • Un service en ligne comme Google Analytics ou QuickBooks ;
  • Une base de données dans le cloud comme Azure SQL Database ;
  • Une base de données ou un fichier sur un ordinateur ou un serveur local dans votre propre organisation.

2 - Power BI Desktop​

Utilisé principalement par les concepteurs affaires et les développeurs BI, Power BI Desktop est une application gratuite que vous pouvez installer sur votre ordinateur personnel, qui vous permet de vous connecter, transformer et visualiser vos données. Avec Power BI Desktop, vous pouvez connecter de multiples sources de données et les combiner (souvent appelé adapter / transformer) dans un modèle de données. Ce modèle vous permet de créer des groupes de visuels que vous pouvez ensuite partager sous forme de rapport avec d'autres personnes dans votre organisation.

La plupart des personnes utilisent Power BI Desktop pour créer des rapports et utilisent ensuite Power BI service pour les partager avec d'autres utilisateurs. Le partage de rapport est disponible seulement pour les détenteurs de licence POWER BI PRO ou PREMIUM.

Les utilisations les plus fréquentes de Power BI Desktop sont les suivantes :

  • Se connecter à des données sources ;
  • Transformer, adapter et nettoyer ces données sources pour créer un modèle ;
  • Créer des visuels, comme des graphiques et des grilles, qui représentent les données visuellement ;
  • Créer des rapports qui sont des groupes / collections de visuels, sur une ou plusieurs pages ;
  • Partager des rapports avec d'autres utilisateurs en utilisant Power BI Service.

3 - Passerelle / "Gateway"​

Power BI Gateway est un logiciel requis pour accéder aux données situées dans un réseau sur site (on premise). Le Gateway agit comme un garde-porte pour la source de données sur site. Si quelqu'un a besoin d'accéder aux données sur site à partir d'une application cloud ou Web, la demande passe par la passerelle. Toutes les demandes de connexion sont traitées par la passerelle et l'accès est accordé en fonction de leur authentification et de leurs accès.

Les passerelles ne transfèrent pas les données de la source sur site vers la plateforme cliente, mais connecte plutôt cette plateforme directement à la source de données sur site. Le client peut ainsi accéder directement aux données depuis son emplacement sur site pour les utiliser et créer des rapports, des tableaux de bord et des analyses de données. La passerelle est utilisée pour faciliter les connexions entre une source de données unique (ou plusieurs sources de données) à la source de données sur site.

Types de passerelle Power BI

Vous pouvez choisir parmi 2 types de passerelles disponibles :

  • Mode standard (recommandé) ;
  • Mode personnel.

Passerelle de données sur site standard (recommandé) :

Cette version de la passerelle de données sur site permet la connexion avec plusieurs sources de données sur site pour plusieurs utilisateurs. Vous pouvez utiliser les données dans Power BI, Azure Analysis Services, Azure Logic Apps, Microsoft Flow, PowerApps, etc. Vous pouvez établir des connexions directes à plusieurs sources de données en installant une seule fois ce type de passerelle de données. Cette passerelle de données est recommandée pour les scénarios complexes où plusieurs utilisateurs doivent accéder à plusieurs sources de données.

Passerelle de données sur site en mode personnel :

Le mode personnel de la passerelle de données sur site est utilisé par un seul utilisateur pour se connecter à différentes sources de données. Il est recommandé lorsqu'une seule personne a besoin d'accéder aux sources de données. Pour créer des rapports et des tableaux de bord à l'aide de Power BI, l'utilisateur ne peut pas partager son privilège d'accès avec d'autres utilisateurs.

La passerelle personnelle s'installe sur le poste de l'utilisateur et limite son utilisation à un usage personnel.

Pour plus de détail, consultez le site : https://data-flair.training/blogs/power-bi-gateway/

Composants du service Power BI (Cloud)​

4 - Service Power BI​

Partie Software as a Service (SaaS) de Power BI.

Il est également connu sous le nom de Power BI Online. Pour accéder au service Power BI, vous devez vous connecter au service Power BI .

Les parties de Power BI

Power BI se compose de plusieurs éléments qui fonctionnent tous ensemble, à commencer par ces trois bases :

  • Une application de bureau Windows appelée Power BI Desktop ;
  • Un service en ligne SaaS ( Software as a Service ) appelé service Power BI ;
  • Des applications mobiles Power BI pour les appareils Windows, iOS et Android.

Pour plus de détail sur Power BI service, consultez : Power BI Service

5 - Portail d'administration​

Interface Web permettant d'administrer les options de Power BI.

Le portail d'administration comprend des paramètres qui régissent Power BI pour tous les utilisateurs de votre organisation. Par exemple, dans le portail d'administration, vous pouvez afficher les métriques d'utilisation, accéder au centre d'administration Microsoft 365 et contrôler la manière dont les utilisateurs interagissent avec Power BI.

Le portail d'administration contrôle les options comme l'impression, le téléchargement des .pbix et l'analyse via Excel.

Le portail d'administration complet est accessible aux administrateurs généraux et aux utilisateurs disposant du rôle d'administrateur Power BI. Si vous n'êtes pas dans l'un de ces rôles, vous ne voyez que les paramètres de capacité dans le portail.

Pour plus d'informations sur le rôle d'administrateur de service Power BI, consultez Présentation du rôle d'administrateur Power BI .

Pour plus d'information sur le portail, consultez le portail d'administration de Power BI

6 - Portail de gestion des passerelles / "Gateway"​

Interface Web permettant de configurer et gérer les accès aux différentes sources de données.

Dans Power BI Desktop, vous êtes connecté directement à votre source de données sur site, par exemple "SQL Server", mais le service Power BI nécessite une passerelle de données pour servir de pont entre le cloud et votre réseau sur site. Le portail de gestion des passerelles permet d'ajouter et de configurer votre source de données locale en tant que source de données accessible dans Power BI Service, via la passerelle. Sans cette configuration, il est impossible de rafraichir des jeux de données ou de faire des connexions directes.

Pour plus de détails : comment configurer une source de données dans la passerelle

7 - Espace de travail / "Workspace"​

Contenant de données principale de Power BI Service, l'espace de travail joue également le rôle d'environnement de travail Web.

L'espace de travail est un environnement partagé pour un groupe de personnes. Vous pouvez avoir plusieurs contenus Power BI dans un espace de travail. Un espace de travail peut contenir des centaines de tableaux de bord, de rapports et d'ensembles de données. Vous pouvez ajouter des personnes (ou un compte Power BI en d'autres termes) à l'espace de travail et ainsi leur donner accès à la modification ou à la lecture du contenu.

Il est possible qu'un compte fasse partie de plusieurs espaces de travail, ou que plusieurs comptes aient accès à un seul espace de travail. Vous pouvez même considérer les espaces de travail comme des dossiers partagés.

Tout le monde a un espace de travail nommé Mon espace de travail qui est similaire à votre dossier "Mes documents" sur votre machine. "Mon espace de travail" ne doit jamais être utilisé pour partager du contenu avec d'autres, sauf pour tester, car il s'agit de votre espace de travail personnel.

Si vous souhaitez partager du contenu avec d'autres, votre point de départ peut être la création d'un autre dossier, qui dans la terminologie Power BI se nommera espace de travail. Les espaces de travail sont appelés « espace de travail d'application » car vous pouvez créer une application basée sur celui-ci.

Il est préférable d'utiliser les espaces de travail comme un environnement collaboratif pour partager du contenu entre les membres d'une équipe.

8 - Jeux de données / "Data set"​

Couche sémantique d'accès aux données sources. Il est possible de partager un même jeu de données entre plusieurs rapports Power BI et plusieurs espaces de travail. Ce concept de jeux de données partagés se nomme en anglais "shared dataset" et jeux de données partagés en français.

Un jeu de données est une collection de données que vous importez ou à laquelle vous vous connectez. Power BI vous permet de vous connecter et d'importer toutes sortes d'ensembles de données et de les rassembler en un seul endroit. Les jeux de données peuvent également générer des données à partir de flux de données.

Les jeux de données sont associés à des espaces de travail et un seul jeu de données peut faire partie de plusieurs espaces de travail. Lorsque vous ouvrez un espace de travail, les jeux de données associés sont répertoriés sous l'onglet "Jeux de données". Chaque jeu de données répertorié représente une source unique de données, par exemple :

  • Un classeur Excel sur OneDrive ;
  • Un jeu de données tabulaire SSAS sur site ;
  • Ou un jeu de données Salesforce.

Il existe de nombreuses sources de données différentes prises en charge et Microsoft en ajoute constamment de nouvelles. Consultez la liste des types de jeux de données que vous pouvez utiliser avec Power BI.

Un jeu de données...

  • Peut-être utilisé dans un ou dans plusieurs espaces de travail.
    • Ceci n'est pas recommandé ;
    • Nous suggérons plutôt d'utiliser les jeux de données partagés.
  • Peut-être utilisé dans de nombreux rapports différents.
  • Peut-être partagé et utilisé dans plusieurs espaces de travail sans être répliqué dans chacun de ces espaces de travail.
    • Les jeux de données partagés sont fortement recommandés et appliqués par défaut par l'équipe de développement.
  • Les visualisations de ces ensembles de données peuvent s'afficher sur de nombreux tableaux de bord différents.

Distinction importante : il est important de comprendre qu'à la base, un jeu de données est toujours un fichier Power BI Desktop. C'est lors de la publication sur Power BI Service qu'un jeu de données est configuré pour devenir:

  1. Un jeu de données partagé lorsque configuré pour être partagé, un même jeu de donnée peut être utilisé pour créer plusieurs rapports provenant de plusieurs Power BI Desktop sans avoir à dupliquer la partie jeux de données.

  2. Un jeu de données unique, combo rapport & jeux de données par défaut, un fichier Power BI Desktop peut contenir le jeu de données et une ou plusieurs pages de rapports. Lors de la publication dans Power BI Service, cela créer un instance ou une copie du jeu de données et du rapport. Si le fichier Power BI Desktop est publié dans un seul espace de travail, cela créer un jeu de données unique qui est utilisé pour un seul rapport

  3. Un jeu de données répliqué dans plusieurs Espaces de travail

    Comme pour le jeu de données unique, un fichier Power BI Desktop contenant et le jeu de données et le rapport, mais dupliqué pour produire plusieurs rapports différents et publié dans plusieurs espaces de travail produira un modèle de données dupliqué.

    Il est fortement recommandé de ne pas utiliser cette méthode et d'utiliser un jeu de données partagé pour créer plusieurs rapports Power BI basé sur le même jeu de données Power BI.

La création de copie de jeux de données Power BI Desktop entraine rapidement plusieurs problématiques:

  • Complexité de maintenir les modèles identiques ;
    • Écart entre différents rapports Power BI Desktop ( code différent entre les jeux de données)
    • Différence de contenue entre les copies du modèle, mesures DAX, tables chargées, etc.
  • Inclusion de contenu inutile dans le rapport Power BI pour un rapport

9 - Rapport [à revoir, Pratique et Norme mélangées]​

Interface principale de présentation des données.

Un rapport Power BI est constitué d'une ou plusieurs pages de visualisations telles que des graphiques en courbes, des cartes et des arborescences. Les visualisations sont également appelées visuels. Toutes les visualisations d'un rapport proviennent d'un seul ensemble de données. Vous pouvez créer des rapports à partir de zéro dans Power BI Desktop et les publier dans le service Power BI, ou Power BI peut les créer lorsque vous vous connectez à des ensembles de données à partir d'Excel, de Power BI Desktop, de bases de données et d'applications SaaS. Par exemple, lorsque vous vous connectez à un classeur Excel contenant des feuilles Power View, Power BI crée un rapport basé sur ces feuilles. Lorsque vous vous connectez à une application SaaS, Power BI importe un rapport prédéfini.

Il existe deux modes d'affichage et d'interaction avec les rapports dans le service Power BI : la vue Lecture et la vue Modification. Lorsque vous ouvrez un rapport, ce dernier s'ouvre par défaut en mode Lecture. Si vous disposez d'autorisations de modification, vous voyez "Modifier le rapport" dans le coin supérieur gauche et vous pouvez afficher le rapport dans la vue Modification. Si un rapport se trouve dans un espace de travail, toute personne disposant d'un rôle d'administrateur, de membre ou de contributeur peut le modifier. Ils ont accès à toutes les fonctionnalités d'exploration, de conception, de création et de partage de la vue modification pour ce rapport. Les personnes avec lesquelles ils partagent le rapport peuvent explorer et interagir avec le rapport en mode Lecture.

Lorsque vous ouvrez un espace de travail, les rapports associés sont répertoriés sous l'onglet contenu. Chaque rapport répertorié représente une ou plusieurs pages de visualisations basées sur un seul des jeux de données sous-jacents. Pour ouvrir un rapport, sélectionnez-le.

Par défaut, le rapport s'ouvre en mode Lecture. Sélectionnez simplement modifier le rapport pour l'ouvrir dans la vue Modification (si vous disposez des autorisations nécessaires).

Un rapport:

  • Se trouve dans un seul espace de travail.
  • Peut-être associé à plusieurs tableaux de bord au sein de cet espace de travail. Les vignettes épinglées à partir de ce rapport peuvent apparaître sur plusieurs tableaux de bord.
  • Peut-être créé à l'aide des données d'un jeu de données. Power BI Desktop peut combiner plusieurs sources de données dans un seul jeu de données pour créer un rapport, et ce rapport peut être importé dans service Power BI.

10 - Tableau de bord / "Dashboard"​

Ensemble de visuels pouvant provenir de plusieurs rapports

Un tableau de bord est quelque chose que vous créez dans le service Power BI ou quelque chose qu'un collègue crée dans le service Power BI et partage avec vous. Il s'agit d'un canevas unique qui contient aucune ou plusieurs tuiles et widgets. Chaque vignette épinglée à partir d'un rapport ou d'un Question&Réponse (Q&R) affiche une visualisation unique qui a été créée à partir d'un ensemble de données et épinglée au tableau de bord. Des pages entières de rapport peuvent également être épinglées à un tableau de bord sous la forme d'une seule vignette. Il existe de nombreuses façons d'ajouter des vignettes à votre tableau de bord ; trop nombreux pour être traités dans ce sujet de présentation. Pour en savoir plus, recherchez "Vignettes du tableau de bord dans Power BI".

Pourquoi les gens créent-ils des tableaux de bord ? Voici quelques-unes des raisons :

  • Pour voir, en un coup d'œil, toutes les informations nécessaires pour prendre des décisions.
  • Pour surveiller les informations les plus importantes sur votre entreprise.
  • Pour s'assurer que tous les collègues sont sur la même longueur d'onde, visualisant et utilisant les mêmes informations.
  • Pour surveiller la santé d'une entreprise, d'un produit, d'une unité commerciale, d'une campagne de marketing, etc.
  • Pour créer une vue personnalisée d'un tableau de bord plus grand - toutes les métriques qui comptent pour vous.

[Pratiques et Normes mélangées] Lorsque vous ouvrez un espace de travail, les tableaux de bord associés sont répertoriés sous l'onglet contenu. Pour ouvrir un tableau de bord, sélectionnez-le en cliquant. Un tableau de bord s'affiche. Chaque tableau de bord représente une vue personnalisée d'un sous-ensemble du ou des ensembles de données sous-jacents. Si vous possédez le tableau de bord, vous aurez également un accès en modification aux ensembles de données et rapports sous-jacents. Si le tableau de bord a été partagé avec vous, vous pourrez interagir avec le tableau de bord et tous les rapports sous-jacents, mais vous ne pourrez pas enregistrer les modifications.

Vous ou un collègue pouvez partager un tableau de bord de différentes manières. Power BI Pro est requis pour partager un tableau de bord et peut être requis pour afficher un tableau de bord partagé.

Un tableau de bord:

  • Est associé à un seul espace de travail
  • Peut afficher des visualisations à partir de nombreux ensembles de données différents
  • Peut afficher des visualisations de nombreux rapports différents
  • Peut afficher des visualisations épinglées à partir d'autres outils (par exemple, Excel)

11- Application Power BI​

Les applications Power BI sont le principal mode de déploiement de rapports et tableaux de bord pour donner accès aux utilisateurs.

Une application est un type de contenu Power BI qui combine des tableaux de bord et des rapports associés, le tout en un seul endroit. Une application peut avoir un ou plusieurs tableaux de bord et un ou plusieurs rapports, tous regroupés. Les applications sont créées par les concepteurs Power BI qui distribuent et partagent les applications avec leurs collègues.

Les applications Power BI Service peuvent être vue comme un groupe de rapports et tableaux de bords préassemblés et publiés pour un groupe d'utilisateurs spécifique. Chacun des rapports et tableaux de bords dans un espace de travail a un paramètre permettant d'indiquer s'il fait partie ou non de l'application.

L'une des meilleures fonctionnalités des applications Power BI Service est qu'il est possible de personnaliser le menu pour organiser le contenu. On peut également y inclure des hyperliens vers de la documentation supplémentaire, d'autre système et application, etc.

Important une application Power BI utilise le contenu d'un seul espace de travail

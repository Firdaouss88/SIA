Détails composants Service PBI

L'architecture fonctionnelle se base principalement sur les entités du service Power BI, mais plusieurs autres entités, tel que les sources de données ont aussi un impact sur l'architecture fonctionnelle mise en place pour un projet spécifique.

Entité de source de données​

Les entités de sources de données regroupent toutes les sources possibles, Web, sur site, du fichier plat au service Web en passant par les bases de données.

Power BI supporte plusieurs sources de données, les règles d'architecture recommandent fortement que toutes les sources de données soient préalablement sécurisées et exposées par un comptoir de données. En d'autres termes, indépendamment de la source de données, cette dernière sera rendue disponible aux utilisateurs par un jeu de données Power BI qui sera connecté sur un comptoir de données.

Afin de faciliter le développement et de supporter les modes exploratoires et le développement Fast Track EODE, plusieurs types de comptoirs sont supportés dans la couche comptoir.

À venir: Insérer un lien vers la documentation sur la couche de comptoir et les types de comptoirs : DM-DML-SBX-SBXU

  1. Source de données cloud
    • Toute source provenant du Web
  2. Source de données sur site
    • Toute source de données sur site
    • Base de données
    • Fichiers plats et Excel

Les sources 1 et 2 peuvent être utilisées en mode exploratoire affaires et exploratoire EODE.

Noter que nous recommandons d'utiliser une source de type comptoir pour charger des données dans Power BI Desktop.

  1. Source de données de comptoir

  • Source officielle des modèles de données Power BI

    Tous les jeux de données certifiés créés avec Power BI Desktop utilisent une source de données de type comptoir ou Sandbox.

Entités de gestion & configuration​

Les entités de gestion sont les interfaces disponibles pour gérer les options du service Power BI ainsi que les sources de données.

Les entités de configuration les plus impactées par l'architecture fonctionnelle sont la console de gestion et les paramètres de passerelles.

  1. Console d'administration : Permets d'activer et désactiver les options du service Power BI

  2. Serveur de passerelle : Serveur sur site sur lequel est installé le Gateway

  3. Console de gestion des passerelles :

    • Interface du service Power BI permettant de configurer des sources de données utilisées dans les rapports.

    • Chaque nouvelle source doit être configurée par cette interface.

    • Requis pour accéder en direct query à une source de données

    • Permets de programmer le rafraichissement des jeux de données

  4. Paramètre de passerelle : Configuration individuelle d'une source de données

    • La console de gestion permet de gérer et configurer plusieurs sources.
    • Celles-ci deviennent des passerelles ou paramètres de passerelle une fois configurées.
    • Chaque nouveau projet d'affaires implique minimalement une nouvelle passerelle.
    • Normalement, une passerelle sera configurée par palier de développement ( INT-ACC-PRD).

Entités du service Power BI (Cloud)​

Le découpage lié à l'architecture fonctionnelle affecte principalement ces entités. C'est avec ces entités que l'on configure les différents scénarios d'architecture fonctionnelle.

  1. Espace de travail : Entité principale de l'architecture fonctionnelle, on utilise les espaces de travail dans différents rôles afin de mettre en place l'architecture fonctionnelle. Ces rôles sont définis au point [Type d'espace de travail](##Type d'espace de travail).
  2. Jeux de données : Un fichier Power BI Desktop, une fois publié dans le service Power BI, génère un jeu de données. Nous recommandons de créer des fichiers Power BI Desktop spécifiques aux jeux de données et de configurer ces derniers en jeux de données partagés.
  3. Rapport: Un fichier Power BI Desktop, une fois publié dans le service Power BI, génère un rapport. Nous recommandons de créer des rapports Power BI basés sur des jeux de données partagés. Dans ce cas, le Fichier Power BI Desktop contient uniquement le rapport.
  4. Tableau de bord : Les tableaux de bord sont des entités créés dans le service Power BI (cloud) et permettent d'assembler des visuels provenant de plusieurs rapports différents.
  5. Application Power BI*: Chaque espace de travail peut créer une application. Les applications offrent une interface simplifiée pour la consommation d'information. Nous recommandons fortement l'utilisation des applications pour publier du contenu aux utilisateurs et limiter l'accès aux espaces de travail aux développeurs, super-utilisateurs et concepteurs affaires.

Types d'espace de travail​

Il existe 2 types d'espace de travail dans le service Power BI. Les espaces de travail personnel et les espaces de travail.

Les espaces travail personnel ont comme vocation un usage personnel sans partage d'information ni collaboration. Nous recommandons d'utiliser les espaces personnels uniquement pour des essais temporaires.

Les espaces de travail ou espace de travail collaboratif sont la base de l'architecture fonctionnelle. Bien qu'il n'existe qu'un seul type d'espace de travail dans le service Power BI, nous décrirons ici plusieurs utilisations des espaces de travail que nous référerons comme les types d'espace de travail.

  1. Dans le service Power BI tous les espaces de travail sont les mêmes et ont les mêmes fonctionnalités. Les types d'espaces de travail sont définis par la nomenclature de l'architecture fonctionnelle.

  2. WS-DS: Espace de travail de jeu de données

    • Créé pour contenir les jeux de données officiels et certifiés de l'équipe de développement BI.
    • Géré par l'équipe de développement BI.
    • Utilisé uniquement dans les projets affaires DGVI (développement corporatif).

  3. WS-RPT : Espace de travail de rapport

    • Créé pour contenir les rapports :

      • Officiels et certifiés;
      • Regroupés par contexte d'affaires;

    • Géré par les responsables de l'espace de travail.

    • Utilisé principalement dans les projets affaires DGVI (développement corporatif).

      Peut être utilisé avec le libre-service ou la conception affaires lorsque des contextes de sécurité sont présents.

  4. WS-LB : Espace de travail de libre-service

    • Créé pour rendre disponible un jeu de données officiel ou certifié à des utilisateurs affaires.
    • Géré par les super-utilisateurs.
    • Supporté par l'équipe de développement BI.
    • Utilisé uniquement dans les projets affaire DGVI incluant du libre-service (super utilisateur).

  5. WS-SBXU: Espace de travail de conception affaires

    • Créé pour offrir aux gens d'affaires un environnement de partage de leur conception.
    • Permet à plusieurs concepteurs affaires de collaborer ensemble.
    • Permet aux concepteurs affaires de partager leur création (rapport) avec des collègues par l'application.
    • Géré par les concepteurs affaires.
    • Utilisé pour offrir aux gens d'affaires un environnement de collaboration pour créer des modèles de données et des rapports de manière entièrement autonome.

  6. WS-SBX: Espace de travail Sandbox DGVI

    1. Créé par l'équipe de développement pour faire des preuves de concept, explorer et tester des opportunités de projets.
    2. Géré par l'équipe de développement.
    3. Utilisé pour déployer des solutions Fast-Track (mettre un lien vers la doc Fast-Track... à venir).

Application Power BI - Notes complémentaires​

Les applications Power BI sont créées à partir d'un espace de travail et sont l'entité de prédilection pour partager du contenu avec les utilisateurs finaux.

  1. Une application est liée à un espace de travail et peut contenir des rapports et tableaux de bord provenant uniquement de cet espace de travail.
    • Dans l'espace de travail, un paramètre permet de sélectionner si oui ou non l'entité est incluse dans l'application lors de la publication.
  2. Tout comme les types d'espaces de travail, les types d'applications sont un concept de l'architecture fonctionnelle.
    • L'application hérite du type de l'espace de travail auquel elle est liée.
  3. Dans les dessins de l'architecture fonctionnelle, le pointillé désigne une option.
    • Les applications présentées dans un cadre pointillé sont optionnelles et seront créées et gérées seulement si requis.
    • Les applications représentées dans un cadre avec ligne continue sont obligatoires. Si on ne requiert pas l'usage de l'application, on ne requiert également pas l'usage de l'espace de travail auquel l'application est liée.

Entités de développement​

Power BI Desktop est le principal outil de développement pour créer des modèles de données et des rapports. Il est également possible d'utiliser Visual Studio, Dax Studio et plusieurs autres outils.

La principale différence entre créer un modèle avec Visual Studio et Power BI Desktop est que pour déployer sur un serveur SSAS, nous devons utiliser Visual Studio.

Actuellement, nous préconisons la conception via Power BI Desktop.

  • Cela nous permet de ne pas mettre en place ni gérer de serveur SSAS.
  • Différents aspects affectent ces choix et ils sont couverts dans un autre document [lien vers l'autre document].

  1. Fichier Power BI Desktop
    • Extension .pbit : Fichier de gabarit.
    • Extension .pbix : Fichier de données: jeux de données & rapports.
  2. .pbit - Gabarit de jeux de données
    • Gabarit créé spécifiquement pour la création de jeux de données.
  3. .pbix - jeux de données
    • Ce fichier ne contient aucune page de rapports.
    • Spécifiquement créé pour générer un jeu de données partagé.
  4. .pbit - Gabarit de rapport
    • Gabarit créé spécifiquement pour la création de visuel et de rapport.
    • Ce gabarit inclut plusieurs configurations pour faciliter la conception des interfaces.
  5. .pbix - Rapport
    • Ce fichier contient seulement des visuels et des pages de rapports et tableaux de bord.
    • Ce fichier ne contient aucune mesure DAX ni modèle de données.
    • Aucune transformation sur les données n'est effectuée dans ce fichier.
  6. .pbit - Gabarit de conception et d'exploration
    • Ce fichier de gabarit est conçu pour faciliter l'exploration de données.
    • Inclut une table de date paramétrable.
    • Inclut plusieurs configurations pour faciliter la conception des interfaces.
    • Inclut des pages de trucs et astuces et des liens vers les formations Power BI dans la vitrine.
  7. pbit - Fichier de conception affaires, d'exploration et de Sandbox
    • Ce fichier inclut un modèle de données et au moins une page de rapports.

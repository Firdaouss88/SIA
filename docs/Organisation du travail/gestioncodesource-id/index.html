Cloner un dépôt dans la machine local​

Avant de cloner le projet, vous devez créer le chemin C:\DEV\Source\Repos pour déposer vos Repos que vous voulez dans votre machine locale.

  1. Ouvrir l'application Visual Studio et cliquer sur cloner un dépôt

  1. Se connecter avec votre compte pour accéder à l'arborescence du repos

  1. Choisir le repos et cliquer sur se connecter et bien vérifier le chemin du repos.

Envoyer / pousser le code de projet à partir d'un référentiel local​

Dès que vous faites une modification sur un fichier ou créer un nouveau dossier ou fichier dans votre référentiel local ( machine locale) qui doit se trouver dans le chemin C:\DEV\Source\Repos , les changements s'afficheront dans l'onglet Modification GIT sur Visual Studio

Sur l'onglet Explorateur de solutions vous avez votre projet qui s'affiche avec son arborescence de documents et dossiers

  1. Après que vous finissez vos modifications, vous devez faire une validation avec un message qui exprime ce que vous avez modifié ou la tâche que vous avez effectué sur votre référentiel local

2) Maintenant vous allez pousser ou de déployer cette validation que vous venez d'effectuer dans le référentiel local dans Git (Azure Devops):

  1. Cliquez sur l'option synchronisation

  1. Cliquez sur Push (envoyer vers votre branche)

Architecture fonctionnelle - configuration par défaut

Architecture fonctionnelle - configuration défaut

Ce document présente les configurations de l'architecture fonctionnelle du service Power BI les plus fréquentes et standards.

Configuration projet affaires DGVI (défaut)​

  1. Créer un espace de travail de jeux de données pour chacun des paliers (INT-ACC-PRD)
    • Assigner le groupe AD de développeur DGVI comme administrateur de l'espace de travail
    • Personne d'autre (groupe AD ou ressource) n'a accès à cet espace de travail
    • Configurer la passerelle de données
  2. Créer un espace de travail de rapport (INT-ACC-PRD)
    • Assigner le groupe AD de développeur DGVI comme administrateur de l'espace de travail
    • Personne d'autre (groupe AD ou ressource) n'a accès à cet espace de travail
  3. Créer un groupe AD pour l'application Power BI
    • Assigner le groupe AD sur l'application Power BI
    • Assigner le groupe AD sur le jeux de données en mode lecture

Configuration projet affaires DGVI avec libre-service (défaut + option LB)​

  1. Configurer l'espace de travail de jeux de données WS-DS, de rapport WS-RPT et l'application comme dans une configuration standard

    • Assigner le groupe de développeur DGVI comme administrateur des espaces de travail

  2. Créer un espace de travail libre-service

    • Assigner le groupe de développeur DGVI comme administrateur de l'espace de travail
    • Créer un groupe AD de tâche super utilisateur pour le sujet du libre-service
    • Assigner le groupe AD de super utilisateur sujet comme membre sur l'espace de travail libre -service WS-LB-sujet
    • Assigner le groupe AD de super utilisateur sujet sur le jeu de données dans le WS-DS et lui assigner le droit Build

  3. Gérer les accès à l'application libre-service (optionnel)

    • Assigner le groupe de développeur DGVI comme administrateur de l'espace de travail

    • Assigner le groupe AD de super utilisateur sujet sur l'application libre-service

    • Au besoin : Créer un groupe AD pour l'application libre-service

      • Assigner le groupe AD de l'application libre-service sur l'application

    • Assigner le groupe AD de l'application des rapports WS-RPT sur l'application de libre-service (optionnel)

      Faire seulement si tous les utilisateurs peuvent et doivent consulter les rapports de libre-service.

      • Cela permet aux utilisateurs de l'application de rapports officiels de consulter les rapports ad-hoc
      • Cette action remplace généralement la création d'un groupe AD spécifique pour gérer l'application de libre-service

Point de validation Qui doit avoir accès aux rapports libre-service :

  • Les super-utilisateurs : Utiliser le groupe AD Super-utilisateur Libre-service pour gérer l'application
  • Peu de gens, différents des super-utilisateurs : Créer un groupe AD spécifique pour l'application libre-service
  • Tout le monde : utiliser le groupe AD de l'application WS-RPT
